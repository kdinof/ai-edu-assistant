# Fact-Check — Верификация контента курсов

Проверяет технические утверждения в сгенерированном контенте курса: существование инструментов, актуальность команд, корректность терминов.

## Когда запускать

- **Автоматически:** после генерации контента модуля на Stage 06 (запускать в фоне через Agent с `run_in_background: true`)
- **Вручную:** по запросу пользователя

## Вход

Путь к файлам контента для проверки. Обычно:
- Один модуль: `courses/{slug}/06-content/module-{NN}/`
- Один урок: `courses/{slug}/06-content/module-{NN}/lesson-{NN}.md`

## Алгоритм

### Шаг 1: Извлечение сущностей

Прочитай все файлы контента и извлеки технические сущности по категориям:

| Категория | Примеры | Приоритет проверки |
|-----------|---------|-------------------|
| **Инструменты и продукты** | Claude Desktop, Claude Cowork, Google Workspace CLI | Высокий |
| **CLI-команды и синтаксис** | `gws auth setup`, `npm install -g @googleworkspace/cli` | Высокий |
| **Библиотеки и пакеты** | `@googleworkspace/cli`, npm-пакеты | Высокий |
| **API и протоколы** | MCP, OAuth, REST API | Средний |
| **Технические термины** | sandbox, agentный режим, Google Cloud проект | Средний |
| **Версии и требования** | macOS 14+, Apple Silicon, Node.js | Средний |
| **Цены и подписки** | Claude Pro $20/мес, Max $100/мес | Низкий (быстро устаревают) |
| **URL и ссылки** | claude.ai/download, nodejs.org | Высокий |

### Шаг 2: Верификация

Для каждой сущности высокого и среднего приоритета:

1. **Web Search** — найти актуальную информацию:
   - Существует ли инструмент/библиотека?
   - Актуальна ли описанная версия/синтаксис?
   - Не был ли инструмент переименован, deprecated или заменён?

2. **Context7 MCP** (если применимо) — проверить документацию:
   - Сначала `resolve-library-id` для поиска библиотеки
   - Затем `query-docs` для проверки конкретных команд/API

3. **Вердикт по каждой сущности:**
   - `✅ VERIFIED` — подтверждено, всё корректно
   - `⚠️ INACCURATE` — существует, но детали неточны (неверный синтаксис, устаревшая версия, неправильное название)
   - `❌ NOT FOUND` — не удалось подтвердить существование
   - `🔍 UNVERIFIABLE` — недостаточно информации для вердикта

### Шаг 3: Отчёт

Сформируй отчёт и сохрани в `courses/{slug}/06-content/module-{NN}/fact-check-report.md`.

Формат отчёта:

```markdown
# Fact-Check Report: Module {NN}

**Дата:** {дата}
**Файлы:** {список проверенных файлов}
**Сущностей проверено:** {число}

## Результат: {PASS | ISSUES FOUND}

## Проблемы (если есть)

### ⚠️ Неточности

| Сущность | Урок | Проблема | Актуальная информация | Источник |
|----------|------|----------|----------------------|----------|
| ... | ... | ... | ... | URL |

### ❌ Не найдено

| Сущность | Урок | Контекст использования | Источник поиска |
|----------|------|----------------------|----------------|
| ... | ... | ... | ... |

## ✅ Проверено и подтверждено

{Список подтверждённых сущностей — кратко}
```

## Важно

- Проверяй именно **фактическую корректность**, не стиль или грамматику
- Для CLI-команд проверяй точный синтаксис (флаги, аргументы, порядок)
- Для npm-пакетов проверяй реальное имя пакета в npm registry
- Для продуктов (Claude Desktop, Cowork) проверяй актуальные названия и функции
- URL проверяй на доступность через WebFetch если возможно
- Цены помечай как `⚠️ CHECK PRICING` — они меняются часто, ставь пометку а не вердикт
- Если Context7 не находит библиотеку, это НЕ значит что она не существует — проверяй через web search дополнительно

## Пример проблемы

Контент содержит: `npm install -g @googleworkspace/cli`

Fact-check обнаруживает: пакет `@googleworkspace/cli` не найден в npm registry. Google Workspace CLI может не существовать как отдельный npm-пакет, или иметь другое имя.

Вердикт: `❌ NOT FOUND` — требует ручной проверки автором курса.
