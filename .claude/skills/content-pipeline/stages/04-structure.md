# Stage 04: Structure

## Задача

Сгенерировать детальную структуру курса в **точном формате** существующих курсов OSNOVA. Результат — `04-structure.md`.

### 2. Формат структуры

Формат OSNOVA:
- **Паспорт курса** — таблица с параметрами
- **Модули** — `### Модуль N: {название} ({XX} мин)`
- **Цель модуля** — bold, одно предложение
- **Уроки** — `**Урок N.M — {название}** ({X} мин)` с буллетами контента
- **Практические уроки** — в каждом модуле
- **Финальный модуль** — итоги + тест
- **Итоговый артефакт** — список deliverables
- **Шаблоны и материалы** — чеклист

### 4. Генерация структуры

**Задачи генерации:**

- [ ] Паспорт курса (таблица)
- [ ] Сквозной проект (описание + таблица вклада по модулям)
- [ ] Модули с уроками (нумерация, длительности, буллеты)
- [ ] Практический урок в каждом модуле
- [ ] Финальный модуль (итоги + тест)
- [ ] Итоговый артефакт (deliverables)
- [ ] Шаблоны и материалы (чеклист)

На основе PRD (модули, scope, outcomes) сгенерируй полную структуру:

**Базовые правила:**
- Каждый модуль: 15-30 минут
- Каждый урок: 5-10 минут
- Нумерация: Модуль N → Урок N.M
- Каждый модуль заканчивается практическим уроком
- Буллеты контента: 3-5 на урок (конкретные, не абстрактные)
- Суммарная длительность = длительность из PRD
- Последний модуль: "Финализация и итоги"

**Правила методологии (обязательные):**

1. **Нарастающая сложность** — модули идут от простого к сложному. Каждый модуль опирается на результат предыдущего. Нельзя проходить модуль 3 без модуля 2. Проверь: если модули можно переставить местами без потери смысла — структура плохая.

2. **Сквозной проект** — финальный артефакт из PRD строится пошагово через все модули. В каждом модуле студент добавляет конкретный элемент к сквозному проекту. Укажи в каждом модуле: "Вклад в сквозной проект: {что добавляется}".

3. **Практика на своих данных** — каждый практический урок содержит задание "сделай это сейчас со своими реальными данными" (не с вымышленными сценариями "Анна из маркетинга"). Студент работает со своими документами, таблицами, письмами.

4. **Максимум 1-2 инструмента** — весь курс строится вокруг одного основного инструмента. Глубина, а не ширина. Если PRD содержит больше инструментов — вынести дополнительные в "бонусный" или "продвинутый" блок.

5. **Безопасность данных (для AI-курсов)** — в каждом модуле, где студент загружает данные в AI, кратко проговори: что безопасно загружать для этой задачи, что нельзя, как анонимизировать при необходимости. Не отдельный модуль — встроенная привычка.

6. **Реалистичные обещания** — не обещай "экономию 80% времени" или "рост зарплаты на 50%". Используй конкретные измеримые примеры: "задача X: было 2 часа → стало 20 минут". Привязывай к конкретной задаче, а не к абстрактному результату.

**Стиль буллетов (из reference):**
- Конкретные примеры (локальные кейсы, Узбекистан где релевантно)
- Формулы и фреймворки
- Антипаттерны ("Главная ошибка: ...")
- Демонстрации ("Демонстрация: ...")

## Quality Checklist

### Формат
- [ ] Формат соответствует reference
- [ ] Паспорт курса заполнен
- [ ] Каждый модуль имеет цель + уроки
- [ ] Каждый урок имеет длительность + буллеты
- [ ] Нумерация последовательная
- [ ] Суммарное время соответствует PRD

### Методология
- [ ] **Нарастающая сложность** — модули нельзя переставить местами без потери смысла
- [ ] **Сквозной проект** — каждый модуль явно указывает вклад в финальный артефакт
- [ ] **Практика на своих данных** — каждый модуль содержит задание на реальных данных студента
- [ ] **1-2 инструмента** — курс не превращается в обзор 5+ разных сервисов
- [ ] **Безопасность (AI-курсы)** — в каждом модуле с загрузкой данных есть правила безопасности
- [ ] **Реалистичные обещания** — экономия привязана к конкретным задачам, а не к абстрактным %

### Контент
- [ ] Практика в каждом модуле
- [ ] Итоговый артефакт описан
- [ ] Шаблоны и материалы перечислены
