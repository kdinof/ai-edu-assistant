# Stage 03: PRD

## Задача

Синтезировать bootstrap + research в Product Requirements Document для курса. Результат — `03-prd.md`.

## Инструкции

### 0. Roadmap

Покажи Roadmap (формат из SKILL.md) с `🔄 03 PRD` как текущим этапом.

### 1. Загрузи контекст

Прочитай:
- `courses/{slug}/01-bootstrap.md`
- `courses/{slug}/02-research.md`
- `foundation/course-design-values.md` (ценности OSNOVA при создании курсов)

### 2. Прочитай шаблон

`.claude/skills/content-pipeline/templates/prd.md`

### 3. Генерация PRD

**Задачи генерации:**

- [ ] Паспорт курса
- [ ] Learning Outcomes (Bloom's Taxonomy)
- [ ] Целевая аудитория
- [ ] Scope (включено / исключено)
- [ ] Топики и модули
- [ ] Практика
- [ ] Безопасность данных (если AI-курс)
- [ ] Инструменты
- [ ] Конкурентный анализ
- [ ] Метрики успеха
- [ ] Риски

Заполни все секции шаблона:

**Паспорт курса** — из bootstrap + уточнения из research

**Learning Outcomes** — переработай из bootstrap, используя:
- Bloom's Taxonomy (конкретные глаголы: создать, применить, анализировать)
- Данные research (что реально нужно ЦА)

**Целевая аудитория** — обогати bootstrap-описание данными из research

**Scope** — определи что включено и что **сознательно исключено** (и почему)

**Топики и модули** — высокоуровневая таблица модулей с примерным временем. Проверь:
- Модули строятся **последовательно** — каждый опирается на результат предыдущего (нарастающая сложность)
- **Сквозной проект** пронизывает все модули — артефакт модуля N является входом для модуля N+1
- Максимум **1-2 основных инструмента** — глубина вместо ширины. Если в bootstrap указано больше, сузь скоуп

**Практика** — каждый модуль должен содержать задание **"сделай это сейчас со своими данными"**, а не только демо-сценарии с вымышленными персонажами

**Безопасность** — если курс про AI-инструменты, включи правила работы с данными: что можно загружать, что нельзя, как анонимизировать. Не отдельный модуль — встроено в каждый модуль по контексту

**Инструменты** — из bootstrap + дополнения из research. Ограничь до 1-2 основных

**Конкурентный анализ** — из research, с чёткой дифференциацией OSNOVA

**Метрики успеха** — предложи реалистичные targets

**Риски** — на основе research и контекста

### 4. Проверка на соответствие ценностям OSNOVA

Убедись что PRD соответствует `foundation/course-design-values.md`:
- **Дофамин, кайф** — есть элементы engagement (практика, быстрые wins)
- **Должен быть результат** — финальный артефакт чёткий и достижимый
- **Модульные навыки** — длительность 1-3 часа
- **Экспертность** — контент основан на research, не поверхностный

### 6. Human Gate 1: PRD Review

Установи state:
```json
{
  "stages": {
    "03-prd": {
      "status": "review_pending"
    }
  },
  "current_stage": "03-prd"
}
```

Покажи пользователю ключевые секции PRD:
- Learning Outcomes
- Scope (включено / не включено)
- Таблицу модулей
- Конкурентный анализ

Покажи Roadmap с `⏳ 03 PRD` (review_pending).

Используй **AskUserQuestion**:
- question: "PRD готов к ревью. Ваше решение?"
- options: `["Утвердить", "Правки", "Отклонить"]`

Если пользователь выбрал "Правки", используй **AskUserQuestion** (freeform, без options):
- question: "Опишите, что нужно изменить в PRD"

При утверждении:
```json
{
  "stages": {
    "03-prd": { "status": "approved", "completed_at": "..." }
  },
  "current_stage": "04-structure"
}
```

При правках — перезапусти генерацию с учётом фидбека. Сохрани фидбек в state:
```json
{
  "stages": {
    "03-prd": { "status": "revision", "feedback": "..." }
  }
}
```
