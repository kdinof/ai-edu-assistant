# Stage 07: Тесты для уроков

## Задача

Сгенерировать тесты для каждого урока в каждом модуле. Тесты проверяют, насколько ученик усвоил материал урока.

## Вход

- `04-structure.md` — learning outcomes модуля
- `06-content/module-{nn}/lesson-{nn}.md` — контент уроков

## Выход

- `07-tests/module-{nn}/lesson-{nn}-test.md` — тест к каждому уроку

## Подход

### Принципы создания тестов

**1. Тест проверяет понимание, а не память**
- Вопросы на применение знаний, а не пересказ фактов
- Вместо "Какие 4 элемента во фреймворке?" → "Вам нужно создать отчёт по продажам. Какой элемент задания вы забыли?" (с примером, где не указан источник данных)

**2. Привязка к реальным ситуациям**
- Каждый вопрос описывает рабочую ситуацию, знакомую ученику
- Варианты ответов — это реальные действия, а не абстракции
- Используй контекст Узбекистана (компании: Uzum, Artel, Humans; валюта: сум)

**3. Правдоподобные дистракторы**
- Неправильные варианты должны быть разумными — это частые заблуждения или ошибки новичков
- Каждый дистрактор содержит зерно истины, но с критической ошибкой
- Нет очевидно абсурдных вариантов

**4. Обучение через пояснения**
- К каждому вопросу — пояснение, ПОЧЕМУ правильный ответ правильный
- Пояснение должно быть кратким (1-2 предложения), но содержать ключевой инсайт
- Если дистрактор базируется на распространённом заблуждении — объясни, в чём ошибка

**5. Разнообразие типов вопросов**
- Используй разные типы для разных целей (см. ниже)
- Не более 2 вопросов одного типа подряд
- Начинай с простого, заканчивай сложным

### Типы вопросов

**Одиночный выбор (single_choice)**
- 4 варианта ответа (A-D)
- 1 правильный
- Лучше всего для: проверки понимания концепций, выбора правильного действия

**Множественный выбор (multiple_choice)**
- 4-5 вариантов ответа
- 2-3 правильных (указать количество в вопросе)
- Лучше всего для: комплексных тем, где важно понимать все составляющие

**Сценарий (scenario)**
- Описание рабочей ситуации + вопрос "что делать?"
- 4 варианта действий
- Лучше всего для: проверки применения навыков

**Найди ошибку (find_error)**
- Пример с ошибкой + вопрос "что здесь не так?"
- 4 варианта
- Лучше всего для: закрепления правильных паттернов, предотвращения типичных ошибок

### Параметры теста

| Параметр | Значение |
|----------|----------|
| Вопросов на урок | 3-5 (зависит от объёма урока) |
| Проходной балл | 70% |
| Минимум типов | 2 разных типа вопросов на тест |

### Калибровка сложности

- **Теоретический урок (Теория):** акцент на понимание концепций → больше single_choice и scenario
- **Практический урок (Практика):** акцент на правильные действия → больше scenario и find_error
- **Смешанный урок:** микс всех типов

### Алгоритм генерации

Для каждого модуля:

1. **Прочитай все уроки модуля** из `06-content/module-{nn}/`
2. **Прочитай learning outcomes модуля** из `04-structure.md`
3. **Для каждого урока:**
   a. Выдели 3-5 ключевых концепций/навыков из урока
   b. Для каждой концепции создай вопрос подходящего типа
   c. Убедись в разнообразии типов вопросов
   d. Проверь, что дистракторы правдоподобны
   e. Напиши пояснения
4. **Сохрани** тест в `07-tests/module-{nn}/lesson-{nn}-test.md`
5. **Проверь качество** по чеклисту (ниже)

### Обработка модулей

- Обрабатывай модули последовательно (module-01 → module-02 → ...)
- После каждого модуля — спроси пользователя: продолжить к следующему или остановиться
- Используй **AskUserQuestion** с вариантами: `["Продолжить", "Остановиться"]`

## Шаблон

Используй `templates/test.md`

### Распределение правильных ответов

**Правильные ответы должны быть равномерно распределены по A, B, C, D** в пределах каждого урока и модуля. Не допускай перекоса, когда большинство ответов — B или C. После генерации теста проверь распределение и перемешай варианты при необходимости.

## Quality Gate — чеклист для самопроверки

Перед сохранением каждого теста проверь:

- [ ] Вопросы проверяют **понимание и применение**, а не запоминание
- [ ] Каждый вопрос привязан к **рабочей ситуации** ученика
- [ ] Дистракторы **правдоподобны** — нет очевидно абсурдных вариантов
- [ ] Пояснения содержат **ключевой инсайт**, а не просто "потому что A правильный"
- [ ] Использованы **минимум 2 типа** вопросов
- [ ] Сложность **нарастает** от первого к последнему вопросу
- [ ] Вопросы **покрывают** ключевые концепции урока
- [ ] Нет вопросов, ответ на которые **не содержится** в материале урока
- [ ] Правильные ответы **равномерно распределены** по A, B, C, D — нет перекоса
