# Stage 02: Deep Research

## Задача

На основе bootstrap-данных провести многоаспектное исследование через Perplexity API с параллельными субагентами. Результат — `02-research.md` с синтезом, паттернами, и quality assessment.

## Инструкции

### 0. Roadmap

Покажи Roadmap (формат из SKILL.md) с `🔄 02 Deep Research` как текущим этапом.

### 1. Анализ Bootstrap

Прочитай `courses/{slug}/01-bootstrap.md`. Извлеки:
- Тема курса
- Целевая аудитория (кто, уровень)
- Инструменты/технологии
- Learning outcomes
- Контекст (локальный рынок, специфика)

### 2. Декомпозиция на аспекты

Разбей тему на **4-6 аспектов** исследования. Обязательные:

1. **Рынок и тренды** — текущее состояние области, тренды 2025-2026, прогнозы
2. **Конкуренты** — существующие курсы по теме. Для каждого курса обязательно:
   - Название и платформа
   - Ссылка на лендинг/страницу курса
   - Цена (или ценовой диапазон)
   - Структура программы (модули, кол-во уроков, длительность)
   - Формат (видео, практика, live)
   - Целевая аудитория курса
   - Позиционирование и ключевое отличие
3. **Целевая аудитория** — для каждой роли/сегмента из bootstrap:
   - Ключевые боли и рутинные проблемы на работе
   - Страхи и барьеры при работе с темой курса
   - Задачи, которые хотят закрыть с помощью обучения
   - Мотивация к обучению

   Обязательно: предложить сегментацию аудитории (2-3 группы) по типу задач или уровню. Например: "рутинные рабочие задачи" vs "психологические/технические барьеры".
4. **Методология обучения** — подходы к дизайну **онлайн-курса** (не внутреннего корпоративного обучения):
   - Структура и формат уроков (длительность, подача материала)
   - Методологии проектирования учебного опыта (Instructional Design)
   - Подходы к вовлечению и удержанию в онлайн-формате
   - Баланс теории и практики
   - Минимум 5-7 различных методологий для сравнения

   ВАЖНО: Фильтровать результаты — нужны методологии для создания онлайн-курса (записанные видео + практика), а НЕ для проведения тренингов, воркшопов, менторства или корпоративных программ.
5. **Инструменты и технологии** — актуальные инструменты, сравнение, что выбрать для курса
6. **Локальный контекст UZ/СНГ** — специфика рынка, адаптация, локальные кейсы, локальные конкуренты

Для каждого аспекта сформулируй **3-5 поисковых запросов** на английском (для лучших результатов Perplexity).

**Рекомендации по запросам для конкретных аспектов:**

- **Конкуренты:** включай запросы типа `"{topic} online course price curriculum modules 2026"`, `"{topic} course syllabus structure pricing comparison"`
- **Целевая аудитория:** используй все роли из bootstrap (не только первую/очевидную). Если в bootstrap указаны "офисные сотрудники", запросы должны покрывать менеджеров, ассистентов, HR, бухгалтеров и т.д. отдельно.
- **Методология:** используй запросы про онлайн-курсы: `"online course instructional design methodology 2026"`, `"e-learning course structure best practices"`, `"lesson design frameworks online education"`, `"learner engagement retention online courses"`. Избегай запросов про "training", "workshop", "corporate learning" — они дают офлайн-результаты.
- **Локальный контекст UZ/СНГ:** включай запросы типа `"AI courses Uzbekistan online education 2026"`, `"обучение ИИ курсы Узбекистан онлайн"`, `"Ustoz AI Yandex education Uzbekistan"`

**Покажи пользователю** план аспектов с запросами.

Используй **AskUserQuestion**:
- question: "План research готов. Запускаем?"
- options: `["Запустить research", "Изменить аспекты/запросы"]`

### 2.5. Пользовательские источники (опционально)

После одобрения аспектов спроси пользователя:

Используй **AskUserQuestion**:
- question: "Есть ли ссылки на курсы/проекты локальных конкурентов (UZ/СНГ), которые нужно проанализировать? Можешь отправить URL лендингов."
- options: ["Нет, продолжаем", "Да, сейчас отправлю ссылки"]

Если пользователь предоставит URL:
1. Используй WebFetch для каждого URL
2. Извлеки: название, цена, структура (модули), позиционирование, аудитория
3. Добавь результаты в аспект "Конкуренты" как отдельный раздел "Локальные конкуренты (UZ/СНГ)"

### 3. Параллельное исследование

После одобрения запусти **отдельный Task() субагент на каждый аспект**.

Каждый субагент:
1. Получает название аспекта и список запросов
2. Для каждого запроса вызывает research.py:

```bash
python .claude/skills/perplexity-research/research.py \
  --topic "{аспект}: {тема курса}" \
  --questions "{запрос 1}" \
  --questions "{запрос 2}" \
  --questions "{запрос 3}" \
  --model sonar-pro \
  --output "courses/{slug}/research-aspects/{aspect_slug}.md" \
  --delay 1.5
```

3. Возвращает: findings (top 3-5), список источников с датами

**Важно:** Все субагенты запускаются параллельно через Task(). Не жди завершения одного перед запуском другого.

Промпт для каждого субагента:

```
Ты исследователь. Твоя задача — провести research по аспекту "{aspect_name}" для курса "{course_name}".

Контекст: Мы создаём записанный онлайн-курс (видеоуроки + практические задания) для платформы онлайн-образования. НЕ тренинг, НЕ воркшоп, НЕ корпоративное обучение.

Запусти research.py с запросами: {queries}

Команда:
python {path_to_research_py} --topic "{aspect}: {course_topic}" --questions {queries_formatted} --model sonar-pro --output "courses/{slug}/research-aspects/{aspect_slug}.md" --delay 1.5

После получения результатов:
1. Прочитай сгенерированный файл
2. Выдели top 3-5 ключевых находок
3. Собери все источники с URL и годом публикации
4. Верни структурированный ответ:
   - Aspect: {name}
   - Queries used: [list]
   - Key findings: [list with details]
   - Sources: [list with url, title, year]
```

### 4. Синтез

После завершения всех субагентов собери результаты и проведи синтез:

**Задачи синтеза:**

- [ ] Ключевые находки — top 3-5 findings на каждый аспект
- [ ] Кросс-аспектные паттерны — темы, которые появляются в 2+ аспектах
- [ ] Противоречия — конфликтующая информация между источниками/аспектами
- [ ] Пробелы — что не удалось найти или покрыто недостаточно
- [ ] Key Insights for Course Design — 5-7 actionable выводов для PRD и структуры

### 5. Проверка качества

Проведи автоматическую проверку:

| Метрика | Порог | PASS | WARN | FAIL |
|---------|-------|------|------|------|
| Аспекты с результатами | 100% | все вернули | — | хоть один пустой |
| Уникальные источники | >= 15 | >= 15 | 10-14 | < 10 |
| Свежие источники (2 года) | >= 60% | >= 60% | 40-59% | < 40% |
| Источников на аспект | >= 2 | все >= 2 | — | хоть один < 2 |

**Verdict:**
- **PASS** — все метрики в норме, можно продолжать
- **WARN** — есть предупреждения, покажи пользователю, предложи дополнительный research или продолжение
- **FAIL** — критические пробелы, предложи повторить research по проблемным аспектам

### 6. Запись результата

Сформируй `courses/{slug}/02-research.md` по шаблону `templates/research-report.md`.

Обнови `pipeline-state.json`:

```json
{
  "stages": {
    "02-research": {
      "status": "completed",
      "completed_at": "{ISO datetime}",
      "metadata": {
        "aspects_count": 5,
        "sources_count": 23,
        "quality_verdict": "PASS"
      }
    }
  },
  "current_stage": "03-prd"
}
```

Удали временную папку `courses/{slug}/research-aspects/` после формирования финального отчёта.
