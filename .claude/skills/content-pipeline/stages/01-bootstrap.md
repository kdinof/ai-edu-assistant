# Stage 01: Bootstrap

## Задача

Собрать всю необходимую информацию об идее курса через интерактивный диалог с пользователем. Результат — заполненный опросник `01-bootstrap.md`.

## Инструкции

### 0. Roadmap

Покажи Roadmap (формат из SKILL.md) с `🔄 01 Bootstrap` как текущим этапом.

### 1. Начало

Прочитай шаблон: `.claude/skills/content-pipeline/templates/bootstrap-questionnaire.md`

Если пользователь уже предоставил идею курса — извлеки из неё максимум информации и заполни то, что можно.

### 2. Интерактивный опрос

Задавай вопросы **по одному блоку за раз**, не все сразу. Используй **AskUserQuestion** для каждого блока.

**Задачи опроса:**

- [ ] 1. **Тема и описание** — Уточни название и суть курса
- [ ] 2. **Целевая аудитория** — Кто будет проходить? Уровень? Что уже знает? Ключевые боли, JTBD (задачи, которые хотят решить), барьеры
- [ ] 3. **Learning Outcomes** — Что студент сможет делать после курса? (помоги сформулировать конкретно)
- [ ] 4. **Финальный артефакт** — Что конкретно студент создаст?
- [ ] 5. **Параметры** — Длительность, формат, количество модулей
- [ ] 6. **Инструменты** — Какие инструменты/софт используются?
- [ ] 7. **Конкуренты** — Есть ли известные аналоги? Чем отличаемся?
- [ ] 8. **Дополнительно** — Эксперт, ограничения, связь с другими курсами

### 3. Помощь пользователю

- Если пользователь затрудняется с Learning Outcomes — предложи 3-4 варианта на основе темы
- Если не знает конкурентов — это ОК, отметь что research покроет это
- Если не уверен в длительности — предложи диапазон на основе темы

### 4. Сохранение

Заполни шаблон `bootstrap-questionnaire.md` ответами пользователя. Сохрани как:

```
courses/{slug}/01-bootstrap.md
```

### 6. Переход

Покажи Roadmap с `✅ 01 Bootstrap` и `⬜ 02 Deep Research` как следующим.

Используй **AskUserQuestion**:
- question: "Bootstrap завершён. Перейти к Deep Research?"
- options: `["Продолжить к Deep Research", "Остановиться"]`
