# Quality Gates — Критерии качества

## Stage 01: Bootstrap

- [ ] Все поля опросника заполнены
- [ ] Целевая аудитория описана конкретно (не "все")
- [ ] Learning outcomes измеримы и проверяемы
- [ ] Длительность реалистична (1-3 часа на курс)
- [ ] Финальный артефакт студента определён

## Stage 02: Deep Research

### Automated Quality Gate

| Метрика | PASS | WARN | FAIL |
|---------|------|------|------|
| Аспекты с результатами | 100% | — | < 100% |
| Уникальные источники | >= 15 | 10-14 | < 10 |
| Свежие источники (2 года) | >= 60% | 40-59% | < 40% |
| Источников на аспект | все >= 2 | — | хоть один < 2 |

### Content Quality
- [ ] 4-6 аспектов исследованы параллельно
- [ ] Покрыты обязательные: рынок, конкуренты, ЦА, методология, инструменты
- [ ] Синтез содержит кросс-аспектные паттерны
- [ ] Противоречия между источниками выявлены и описаны
- [ ] Пробелы в данных идентифицированы
- [ ] Key Insights for Course Design (5-7 actionable выводов)

### Verdict
| Verdict | Условие | Действие |
|---------|---------|----------|
| **PASS** | Все метрики в норме | Продолжать к Stage 03 |
| **WARN** | Есть предупреждения | Показать пользователю, предложить доп. research или продолжение |
| **FAIL** | Критические пробелы | Повторить research по проблемным аспектам |

## Stage 03: PRD

- [ ] Все секции шаблона заполнены
- [ ] Learning outcomes конкретны и измеримы (по Bloom's Taxonomy)
- [ ] Scope чётко ограничен (что включено / что НЕ включено)
- [ ] Целевая аудитория сегментирована
- [ ] Конкуренты проанализированы с дифференциацией OSNOVA
- [ ] Метрики успеха определены
- [ ] Соответствует ценностям из `foundation/course-design-values.md`:
  - Дофамин и кайф (engagement)
  - Должен быть результат (completion + assessment)
  - Модульные навыки (1-3 часа)
- [ ] **Сквозной проект** определён — артефакт строится пошагово через все модули
- [ ] **Максимум 1-2 основных инструмента** — глубина, не ширина
- [ ] **Практика на своих данных** прописана как требование
- [ ] **Безопасность данных** (для AI-курсов) — встроена в каждый модуль

## Stage 04: Structure

### Формат
- [ ] Формат соответствует шаблону `templates/course-structure.md`
- [ ] Паспорт курса заполнен (таблица)
- [ ] Каждый модуль имеет: название, цель, уроки
- [ ] Каждый урок имеет: название, длительность, буллеты контента
- [ ] Нумерация последовательная (Module X → Lesson X.Y)
- [ ] Суммарная длительность соответствует PRD
- [ ] Финальный модуль включает итоги + тест

### Методология
- [ ] **Нарастающая сложность** — модули нельзя переставить без потери смысла
- [ ] **Сквозной проект** — каждый модуль указывает вклад в финальный артефакт
- [ ] **Практика на своих данных** — каждый модуль содержит задание на реальных данных студента
- [ ] **1-2 инструмента** — курс не обзор 5+ сервисов, а глубокое освоение одного
- [ ] **Безопасность (AI-курсы)** — правила работы с данными в каждом модуле с загрузкой файлов
- [ ] **Реалистичные обещания** — экономия привязана к конкретным задачам, не абстрактные %

### Контент
- [ ] Есть практический урок в каждом модуле
- [ ] Итоговый артефакт курса описан
- [ ] Шаблоны и материалы перечислены

## Stage 07: Tests

- [ ] Вопросы проверяют **понимание и применение**, а не запоминание фактов
- [ ] Каждый вопрос привязан к **рабочей ситуации** ученика
- [ ] Дистракторы **правдоподобны** — нет очевидно абсурдных вариантов
- [ ] Пояснения содержат **ключевой инсайт**, а не просто "A правильный"
- [ ] Использованы **минимум 2 типа** вопросов на тест (single_choice, multiple_choice, scenario, find_error)
- [ ] Сложность **нарастает** от первого к последнему вопросу
- [ ] Вопросы **покрывают** ключевые концепции урока
- [ ] Нет вопросов, ответ на которые **не содержится** в материале урока
- [ ] Тест к каждому уроку содержит **3-5 вопросов**

## Human Gate 1: PRD Review

**Что проверяет reviewer:**
- Соответствие идее курса из bootstrap
- Реалистичность learning outcomes
- Полнота scope
- Правильность позиционирования относительно конкурентов

## Human Gate 2: Методолог Review

**Что проверяет методолог:**
- Логичность последовательности модулей
- Достаточность практики в каждом модуле
- Реалистичность тайминга уроков
- Качество learning outcomes на уровне модулей
- Соответствие формату OSNOVA
