## Вопросы

### Вопрос 1

**Тип:** single_choice

Что такое skill в Cowork?

- A) Встроенная функция Google Workspace CLI для автоматизации
- B) Сохранённая инструкция для задачи, которую вы делаете регулярно — запускается одной фразой
- C) Плагин, который нужно скачать и установить отдельно
- D) Режим работы Cowork, в котором он выполняет задачи быстрее

**Правильный ответ:** B
**Пояснение:** Skill — это файл с инструкцией для повторяющейся задачи. Вы сохраняете удачное задание один раз (через Settings → Skills), а потом запускаете его одной фразой. Результат стабильный каждый раз, без необходимости формулировать задание заново.

---

### Вопрос 2

**Тип:** scenario

Вы каждую пятницу тратите 3-5 минут на формулировку одного и того же задания для еженедельного отчёта. Иногда забываете указать папку — и Cowork сохраняет документ в корень Drive. Как решить эту проблему?

- A) Записать задание в блокнот и копировать каждый раз
- B) Создать skill «Пятничный отчёт» с полной инструкцией — запускать одной фразой
- C) Попросить Cowork запомнить задание навсегда
- D) Использовать другой AI-инструмент с функцией «сохранить промпт»

**Правильный ответ:** B
**Пояснение:** Skill решает обе проблемы: не нужно формулировать задание заново (экономия 3-5 минут) и все элементы (включая папку сохранения) зафиксированы в инструкции. Одна фраза «Пятничный отчёт» — и Cowork выполняет по сохранённой инструкции.

### Вопрос 3

**Тип:** find_error

Коллега создал skill для каждого действия: «Ответить на письмо клиенту», «Написать сообщение в чат», «Найти файл на Drive», «Ответить на звонок». Что не так с этим подходом?

- A) Слишком много skills — в Cowork ограничение на 5 штук
- B) Skills создаются для задач с повторяющейся структурой — ответы на письма и звонки каждый раз уникальны, skill для них не нужен
- C) Нужно объединить все skills в один общий skill «Рабочий день»
- D) Skills должны быть только для задач с Google Sheets — документы и письма не поддерживаются

**Правильный ответ:** B
**Пояснение:** Skill полезен, когда структура задачи повторяется, а меняются только данные (еженедельный отчёт, подготовка к планёрке). Ответ на письмо каждый раз уникален — другой адресат, тема, контекст. Для таких задач skill — это лишняя работа.

### Вопрос 4

**Тип:** scenario

Вы придумали skill для анализа квартальных расходов — но ни разу не пробовали это задание на практике. Стоит ли сразу создать skill?

- A) Да — чем раньше создадите, тем быстрее начнёте экономить время
- B) Нет — сначала выполните задание вручную, убедитесь что результат хороший, и только потом сохраните как skill
- C) Да — Cowork исправит ошибки в skill автоматически при первом запуске
- D) Нет — skills создаются только через Settings, а не через задания

**Правильный ответ:** B
**Пояснение:** Правило хороших skills: сохраняйте только задания, которые уже дали хороший результат. Если вы не проверили задание на практике, в skill может быть ошибка (неправильный источник данных, неподходящий формат) — и вы будете получать плохой результат каждый раз.
