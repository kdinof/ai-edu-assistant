## Вопросы

### Вопрос 1

**Тип:** single_choice

Чем составной skill отличается от простого?

- A) Составной skill использует другой формат файла
- B) Составной skill содержит цепочку из нескольких шагов, где результат каждого передаётся в следующий
- C) Составной skill может работать только с Google Sheets, а простой — с любыми инструментами
- D) Составной skill стоит дороже в использовании

**Правильный ответ:** B
**Пояснение:** Составной skill — это пайплайн, сохранённый для повторного использования. Он содержит 2-4 задачи в цепочке (например: данные из Sheets → отчёт в Docs → презентация в Slides). Технически это тот же skill, но с несколькими шагами в инструкции.

### Вопрос 2

**Тип:** scenario

Вы хотите, чтобы Cowork каждый понедельник в 9:00 автоматически готовил повестку к планёрке. Вы настроили scheduled task. Но в понедельник утром ваш Mac был в режиме сна до 9:30. Что произойдёт?

- A) Cowork выполнит задачу в облаке и покажет результат, когда вы откроете Mac
- B) Задача пропустится — scheduled tasks работают только при включённом Mac и открытом Claude Desktop
- C) Cowork отправит вам уведомление и выполнит задачу, когда Mac проснётся
- D) Задача выполнится в 9:00 следующего дня

**Правильный ответ:** B
**Пояснение:** Scheduled tasks — локальная автоматизация, не облачный сервис. Если Mac в режиме сна или Claude Desktop закрыт — задача пропускается. Практический совет: для утренних задач убедитесь, что Mac и Claude Desktop запущены до нужного времени, или запустите skill вручную.

### Вопрос 3

**Тип:** find_error

Коллега создал один skill «Полный рабочий день» из 12 шагов: утренний отчёт, проверка почты, анализ таблицы, 3 письма клиентам, подготовка к встрече, протокол, вечерняя сводка, обновление трекера, отправка отчёта, резервное копирование. Что не так?

- A) В Cowork нет ограничения на количество шагов — можно оставить как есть
- B) Skill слишком сложный — лучше разбить на 3-4 отдельных skill (утро, встреча, вечер), которые проще запускать и отлаживать
- C) Нужно добавить ещё шагов, чтобы покрыть все рабочие задачи
- D) Skill должен содержать только один шаг — составные skills не поддерживаются

**Правильный ответ:** B
**Пояснение:** Skill из 10+ шагов слишком сложен для отладки — если один шаг даёт неправильный результат, разбираться долго. Лучше разбить на 2-3 отдельных skill (утренний, для встречи, вечерний). Проще запустить два skill по очереди, чем отлаживать один мегасложный.

### Вопрос 4

**Тип:** single_choice

Как правильно распределить информацию между CLAUDE.md и skills?

- A) Всё записать в CLAUDE.md — skills не нужны
- B) CLAUDE.md — контекст (должность, правила, формат документов), skills — конкретные инструкции для задач
- C) CLAUDE.md — для личных заметок, skills — для рабочих задач
- D) Продублировать информацию в обоих местах для надёжности

**Правильный ответ:** B
**Пояснение:** CLAUDE.md и skills дополняют друг друга: CLAUDE.md хранит контекст (кто вы, правила компании, формат документов), а skills — конкретные шаги выполнения задач. Cowork использует оба: из CLAUDE.md берёт контекст, из skill — инструкции. Дублировать не нужно.
