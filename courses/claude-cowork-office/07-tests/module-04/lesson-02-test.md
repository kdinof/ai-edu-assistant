## Вопросы

### Вопрос 1

**Тип:** single_choice

Что такое пайплайн в контексте работы с Cowork?

- A) Отдельная программа для автоматизации задач в Google Workspace
- B) Цепочка задач, где результат каждого шага становится входом для следующего: данные → анализ → отчёт → презентация
- C) Функция Google Sheets для автоматического обновления данных
- D) Набор промптов, которые нужно отправлять Cowork по одному

**Правильный ответ:** B
**Пояснение:** Пайплайн — цепочка связанных задач. Cowork читает данные из Sheets, анализирует, создаёт отчёт в Docs и презентацию в Slides. Каждый шаг опирается на предыдущий, и всё описывается одним заданием.

### Вопрос 2

**Тип:** find_error

Коллега работает с Cowork пошагово: «Проанализируй таблицу продаж» → ждёт → «Теперь создай отчёт на основе анализа» → ждёт → «Теперь сделай презентацию из отчёта». Что можно улучшить?

- A) Ничего — пошаговый подход правильный, так легче контролировать результат
- B) Описать весь пайплайн в одном задании — Cowork увидит конечную цель и лучше спланирует промежуточные шаги
- C) Нужно запускать три разных сессии Cowork для каждого шага
- D) Нужно добавить паузы между шагами для лучшего результата

**Правильный ответ:** B
**Пояснение:** Когда весь пайплайн описан в одном задании, Cowork видит конечную цель и лучше планирует шаги. Он знает, что данные анализа пойдут в отчёт, а тезисы из отчёта — в презентацию. Вы проверяете план один раз вместо трёх.

### Вопрос 3

**Тип:** scenario

Вы проверяете план Cowork для пайплайна «данные → отчёт → презентация». В плане 3 шага, но во втором шаге (создание отчёта) Cowork заново читает таблицу Google Sheets вместо использования результатов анализа из шага 1. Что делать?

- A) Одобрить план — Cowork перечитает данные быстро, это не влияет на результат
- B) Скорректировать план до одобрения: попросить Cowork использовать результаты шага 1 для отчёта, а не читать таблицу заново
- C) Отменить задание и дать три отдельных задания вместо пайплайна
- D) Одобрить план и потом вручную заменить данные в отчёте

**Правильный ответ:** B
**Пояснение:** Важно проверять связи между шагами плана: данные из шага 1 должны использоваться в шаге 2, а не читаться заново. Скорректировать план до одобрения проще, чем переделывать результат. Это одна из четырёх проверок плана пайплайна.

### Вопрос 4

**Тип:** scenario

Вам нужно написать одно короткое благодарственное письмо партнёру за успешную встречу. Стоит ли оформлять это как пайплайн?

- A) Да — любую задачу лучше оформлять как пайплайн для максимальной эффективности
- B) Нет — это разовая простая задача, достаточно одного задания без цепочки
- C) Да — нужно сначала проанализировать данные о встрече, потом создать письмо, потом презентацию с итогами
- D) Нет — Cowork не умеет создавать письма, это задача для чат-бота

**Правильный ответ:** B
**Пояснение:** Пайплайн полезен для цепочек из нескольких связанных шагов с общим источником данных. Одно короткое письмо — разовая простая задача, для которой достаточно обычного задания по фреймворку «Что → Откуда → Куда → Формат».
