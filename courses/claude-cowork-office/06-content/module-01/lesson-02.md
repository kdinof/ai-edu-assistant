## Видео-скрипт

### Вступление (30 сек)

В прошлом уроке мы разобрались, чем агент отличается от чат-бота. Теперь давайте настроим всё, что нужно для работы: установим Claude Desktop, включим Cowork и подключим Google Workspace CLI — официальный инструмент от Google. К концу этого урока Claude будет видеть ваши Google Docs и Sheets.

### Основная часть

**Что вам понадобится**

Перед началом убедитесь, что у вас есть:
- Mac с процессором Apple Silicon (M1, M2, M3 или новее)
- macOS 14 (Sonoma) или новее
- Подписка Claude Pro ($20/мес) или Max ($100/мес) — Cowork недоступен на бесплатном плане
- Google аккаунт (бесплатный Google Workspace достаточен)

**Шаг 1: Установка Claude Desktop (2 мин)**

1. Откройте сайт claude.ai/download
2. Нажмите «Download for macOS»
3. Откройте скачанный файл
4. Перетащите Claude.app в папку Applications
5. Запустите Claude Desktop из Applications
6. Войдите в свой аккаунт с подпиской Pro или Max

После входа вы увидите три вкладки вверху окна: **Chat**, **Cowork** и **Code**. Нажмите на **Cowork** — это и есть агентный режим.

**Шаг 2: Установка Google Workspace CLI (3 мин)**

Google Workspace CLI — это официальный инструмент от Google (github.com/googleworkspace/cli). Он позволяет работать с Google Docs, Sheets, Drive, Gmail и Calendar через командную строку. Claude Cowork будет использовать его для доступа к вашим файлам.

Откройте Terminal (приложение «Терминал» на Mac) и выполните одну команду:

```bash
npm install -g @googleworkspace/cli
```

Если npm не установлен, сначала установите Node.js с сайта nodejs.org.

Проверьте установку:
```bash
gws --version
```

Если видите номер версии — всё установлено.

**Шаг 3: Авторизация Google аккаунта (3 мин)**

Теперь нужно подключить ваш Google аккаунт. В Terminal выполните:

```bash
gws auth setup
```

Эта команда сделает всё автоматически:
- Создаст Google Cloud проект для вас
- Включит нужные API (Drive, Docs, Sheets)
- Откроет браузер для авторизации

Следуйте инструкциям в браузере: выберите Google аккаунт и разрешите доступ. Это одноразовая процедура.

После этого выполните:
```bash
gws auth login
```

Выберите нужные сервисы (Drive, Docs, Sheets) — и готово.

**Шаг 4: Проверка подключения (1 мин)**

Проверьте, что `gws` видит ваши файлы:

```bash
gws drive files list --params '{"pageSize": 5}'
```

Если видите список ваших файлов в Google Drive — подключение работает.

Теперь перейдите в Claude Desktop → вкладка Cowork и напишите:
> «Покажи мои последние файлы в Google Drive, используя команду gws»

Claude должен выполнить команду и показать список файлов.

**Безопасность данных**

Важный момент: через Google Workspace CLI Claude получает доступ к вашему Google аккаунту. Вот что нужно знать:

- Claude Cowork работает в изолированной среде (sandbox) — данные не уходят за пределы вашего сеанса
- `gws` использует стандартную OAuth-авторизацию Google — тот же механизм, что и мобильные приложения
- Вы контролируете, к каким сервисам `gws` имеет доступ (выбираете при `gws auth login`)
- **Что безопасно подключать:** рабочие документы, отчёты, таблицы с агрегированными данными
- **Что НЕ стоит подключать:** документы с персональными данными клиентов (имена + телефоны + паспортные данные), финансовая отчётность с NDA, пароли и ключи доступа

Рекомендация: создайте отдельную папку в Google Drive для работы с Claude и храните в ней только те файлы, которые безопасно анализировать через AI.

### Заключение (30 сек)

Отлично! У вас теперь настроен Claude Cowork с подключённым Google Workspace. Обратите внимание, как просто это было — две команды в Terminal, и Claude уже видит ваши документы. В следующем уроке мы создадим CLAUDE.md — файл, который сделает Claude вашим персональным ассистентом, а не просто универсальным AI.

## Текстовое сопровождение

### Требования

| Параметр | Требование |
|----------|-----------|
| Компьютер | Mac с Apple Silicon (M1/M2/M3+) |
| ОС | macOS 14 (Sonoma) или новее |
| Подписка | Claude Pro ($20/мес) или Max ($100/мес) |
| Google | Бесплатный Google аккаунт (Workspace не обязателен) |
| Node.js | Для установки gws CLI через npm |

### Пошаговая инструкция

#### 1. Установка Claude Desktop

1. Перейдите на claude.ai/download
2. Скачайте версию для macOS
3. Перетащите Claude.app → Applications
4. Запустите и войдите в аккаунт
5. Выберите вкладку **Cowork** вверху окна

#### 2. Установка Google Workspace CLI

В Terminal:
```bash
# Установить gws CLI
npm install -g @googleworkspace/cli

# Проверить установку
gws --version
```

Альтернативные способы установки:
```bash
# Через Rust (cargo)
cargo install --git https://github.com/googleworkspace/cli --locked

# Через Nix
nix run github:googleworkspace/cli
```

#### 3. Авторизация

```bash
# Первичная настройка (создаёт Google Cloud проект автоматически)
gws auth setup

# Авторизация с выбором сервисов
gws auth login
```

При `gws auth login` выберите сервисы, которые хотите подключить:
- Google Drive (файлы и папки)
- Google Docs (документы)
- Google Sheets (таблицы)
- Google Calendar (по желанию)
- Gmail (по желанию)

#### 4. Проверка

```bash
# Проверить доступ к Drive
gws drive files list --params '{"pageSize": 5}'
```

Затем в Claude Desktop → Cowork:
> «Покажи мои файлы в Google Drive через gws»

Если Claude показывает список файлов — настройка завершена.

### Сравнение: почему gws, а не сторонние инструменты

| Параметр | Google Workspace CLI (gws) | Сторонние MCP-серверы |
|----------|---------------------------|----------------------|
| Разработчик | Google (официальный) | Сообщество |
| Установка | 1 команда (`npm install`) | Клонирование репо, настройка env |
| Настройка OAuth | Автоматическая (`gws auth setup`) | Ручная (Google Cloud Console) |
| Поддерживаемые API | Все Google Workspace API | Ограниченный набор |
| Обновления | От Google | Зависит от автора |

### Безопасность данных

| Безопасно | Не рекомендуется |
|-----------|-----------------|
| Рабочие отчёты | Персональные данные клиентов |
| Агрегированные таблицы | Финансовая отчётность с NDA |
| Шаблоны документов | Пароли и ключи доступа |
| Черновики писем | Документы с медицинскими данными |

**Рекомендация:** создайте отдельную папку в Google Drive для работы с Claude.

### Устранение проблем

| Проблема | Решение |
|----------|---------|
| `npm: command not found` | Установите Node.js с nodejs.org |
| `gws: command not found` | Перезапустите Terminal после установки |
| Ошибка авторизации | Повторите `gws auth login` |
| Claude не видит файлы | Проверьте `gws drive files list` в Terminal |
| Нет доступа к Docs/Sheets | При `gws auth login` выберите нужные сервисы |

## Ключевые выводы

1. **Cowork требует Mac** (Apple Silicon, macOS 14+) и **подписку Pro или Max**. На бесплатном плане агентный режим недоступен
2. **Google Workspace CLI** — официальный инструмент от Google. Установка: `npm install -g @googleworkspace/cli`, авторизация: `gws auth setup` → `gws auth login`. Проще, чем настройка сторонних MCP-серверов
3. **Безопасность:** Claude работает в sandbox, а `gws` использует стандартную OAuth-авторизацию Google. Создайте отдельную папку для работы с AI и не подключайте документы с персональными данными
