## Видео-скрипт

### Вступление (30 сек)

Здравствуйте! В предыдущих модулях вы научились создавать документы в Google Docs и работать с таблицами в Google Sheets. Но есть ещё один формат, без которого не обходится ни одна рабочая неделя — презентации. В этом уроке разберём, как Cowork создаёт презентации и что у него получается хорошо, а что — нет.

### Основная часть

**Два подхода к презентациям**

У Cowork есть два способа создать презентацию:

**Подход 1: Google Slides через Google Workspace CLI.** Cowork создаёт презентацию напрямую в Google Slides — добавляет слайды, текст, таблицы. Результат — готовый файл в Google Drive, который можно сразу открыть и показать.

Плюсы: файл сразу в Google Slides, можно совместно редактировать, не нужно ничего скачивать.
Ограничения: базовое оформление — нет сложного дизайна, анимаций, нестандартных макетов.

**Подход 2: PPTX через code execution.** Cowork генерирует PowerPoint-файл программно — через Python-библиотеку `python-pptx`. Даёт больше контроля: точное размещение элементов, цвета, шрифты, графики.

Плюсы: более гибкий дизайн, возможность добавить графики и точно настроить оформление.
Ограничения: файл нужно скачать и открыть в PowerPoint или Google Slides. Нет совместного редактирования до загрузки.

**Какой подход выбрать?**

Простое правило:
- **Рабочие презентации** (статус по проекту, отчёт для руководителя, повестка встречи) → Google Slides через gws. Быстро, удобно, всегда под рукой.
- **Презентации для внешней аудитории** (клиент, партнёр, конференция) → PPTX, если важен дизайн. Или Google Slides + ручная доработка в редакторе.

В большинстве рабочих ситуаций — Google Slides через gws.

**Как ставить задачу на создание презентации**

Используйте тот же фреймворк из модуля 2 — Что → Откуда → Куда → Формат:

Пример задания:
> «Создай презентацию в Google Slides на 8 слайдов по результатам квартального отчёта. Возьми данные из Google Sheets "Продажи_Q1_2026" и текст из Google Docs "Отчёт Q1". Структура: титульный слайд, ключевые метрики, динамика по месяцам, топ-5 продуктов, проблемы, план на Q2, выводы. Сохрани в папку "Презентации" в Google Drive.»

Обратите внимание: вы указали структуру слайдов, источники данных и папку — Cowork не будет гадать.

**Демонстрация: презентация из отчёта**

Допустим, у вас в Google Docs есть готовый квартальный отчёт, а в Google Sheets — таблица с цифрами. Вы даёте Cowork задание создать презентацию для руководства.

Что происходит:
1. Cowork читает отчёт из Google Docs
2. Читает данные из Google Sheets
3. Создаёт презентацию в Google Slides: титульный слайд, таблица с метриками, выводы, рекомендации
4. Добавляет speaker notes к каждому слайду — тезисы для выступления
5. Сохраняет в указанную папку

Результат: готовая презентация из двух источников данных за 2-3 минуты.

**Что Claude делает хорошо и что — нет**

| Хорошо | Не очень |
|--------|----------|
| Структура и логика слайдов | Сложный визуальный дизайн |
| Ключевые тезисы (не перегружает текстом) | Анимации и переходы |
| Таблицы с данными | Нестандартные макеты |
| Speaker notes | Инфографика |
| Объединение данных из Sheets и Docs | Брендированные шаблоны |

Важно: Claude создаёт **контент** презентации, а не дизайн. Если вам нужно красиво оформить — используйте корпоративный шаблон в Google Slides и попросите Cowork заполнить его содержимым.

### Заключение (30 сек)

Итак, для рабочих презентаций используйте Google Slides через gws — быстро и удобно. Claude отлично справляется со структурой, тезисами и speaker notes, но дизайн лучше доработать вручную или использовать готовый шаблон. В следующем уроке мы соберём всё вместе: данные из Sheets → отчёт в Docs → презентация — в один пайплайн.

## Текстовое сопровождение

### Два подхода к презентациям

| Параметр | Google Slides (gws) | PPTX (code execution) |
|----------|---------------------|----------------------|
| Формат | Google Slides в Drive | Файл .pptx |
| Дизайн | Базовый | Более гибкий |
| Доступ | Сразу в браузере | Нужно скачать |
| Совместная работа | Да | После загрузки в Drive |
| Лучше для | Рабочие презентации | Презентации для клиентов |

### Когда какой подход

- **Google Slides (gws):** статус-отчёт, повестка встречи, квартальный отчёт для руководства — любая презентация, которую нужно быстро создать и сразу показать
- **PPTX:** презентация для клиента, конференции или партнёра — когда важен визуальный дизайн

### Пример задания

```
Создай презентацию в Google Slides на 8 слайдов
по результатам квартального отчёта.

Источники:
- Google Sheets "Продажи_Q1_2026" — цифры и метрики
- Google Docs "Отчёт Q1" — выводы и рекомендации

Структура слайдов:
1. Титульный: название, период, автор
2. Ключевые метрики: выручка, сделки, средний чек
3. Динамика по месяцам: таблица январь-март
4. Топ-5 продуктов по выручке
5. Проблемы и риски
6. План на Q2
7. Выводы и рекомендации
8. Вопросы

К каждому слайду — speaker notes (2-3 предложения).
Сохрани в папку "Презентации" в Google Drive.
```

### Сильные и слабые стороны Claude в презентациях

**Сильные:**
- Логичная структура слайдов
- Краткие тезисы (не перегружает текстом)
- Таблицы с данными из Google Sheets
- Speaker notes для выступления
- Объединение данных из нескольких источников

**Слабые:**
- Нет сложного дизайна (градиенты, нестандартные макеты)
- Нет анимаций и переходов между слайдами
- Не работает с брендированными шаблонами автоматически
- Инфографика и сложные графики — ограничены

**Рекомендация:** используйте корпоративный шаблон → попросите Cowork заполнить содержимым.

## Ключевые выводы

1. **Два подхода:** Google Slides через gws (быстро, для рабочих задач) и PPTX через code execution (больше контроля над дизайном, для внешних презентаций). В большинстве случаев Google Slides — оптимальный выбор
2. **Задание = Что → Откуда → Куда → Формат.** Укажите структуру слайдов, источники данных (Sheets, Docs) и папку для сохранения — Claude создаст презентацию из нескольких источников за минуты
3. **Claude создаёт контент, а не дизайн.** Структура, тезисы, таблицы, speaker notes — отлично. Сложный визуальный дизайн — доработайте вручную или используйте корпоративный шаблон
