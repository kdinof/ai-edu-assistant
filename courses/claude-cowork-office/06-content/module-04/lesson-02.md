## Видео-скрипт

### Вступление (30 сек)

Здравствуйте! До сих пор мы использовали Cowork для отдельных задач: создать документ, проанализировать таблицу, сделать презентацию. Но настоящая сила агента — в цепочках задач, где результат одного шага становится входом для следующего. Это называется пайплайн.

### Основная часть

**Что такое пайплайн**

Пайплайн — цепочка задач, где каждый шаг опирается на предыдущий:

1. **Данные** (Google Sheets) → 2. **Анализ** (новый лист с выводами) → 3. **Отчёт** (Google Docs) → 4. **Презентация** (Google Slides)

Без Cowork вы делаете каждый шаг вручную: открываете таблицу, копируете цифры в документ, форматируете, потом копируете тезисы в презентацию. Это 4 инструмента и 30-60 минут работы.

С Cowork — одно задание. Он делает всю цепочку через Google Workspace CLI: читает данные, анализирует, создаёт документ, формирует презентацию. Результат — 4 связанных артефакта.

**Демонстрация: полный цикл**

Представьте задачу: подготовить квартальный отчёт для руководства. У вас есть таблица продаж в Google Sheets.

Вот как выглядит задание для Cowork:

> «Подготовь пакет документов по итогам Q1 2026:
> 1. Открой Google Sheets "Продажи_Q1_2026". Проанализируй данные: общая выручка, динамика по месяцам, топ-5 продуктов, сравнение с Q4 2025. Создай лист "Анализ Q1" с результатами.
> 2. На основе анализа создай отчёт в Google Docs в папке "Отчёты": ключевые метрики (таблица), 3 главных вывода, рекомендации на Q2.
> 3. Из отчёта создай презентацию в Google Slides на 7 слайдов в папке "Презентации": титульный, метрики, динамика, топ-продукты, выводы, план Q2, вопросы. К каждому слайду — speaker notes.»

Одно задание — три артефакта:
- Лист «Анализ Q1» в Google Sheets
- Отчёт в Google Docs
- Презентация в Google Slides

Всё связано: цифры в презентации совпадают с отчётом, потому что взяты из одного источника.

**Одно задание vs пошаговые команды**

Важное правило: описывайте весь пайплайн в одном задании, а не давайте команды по одной.

| Подход | Пример | Результат |
|--------|--------|-----------|
| По одной команде | «Сначала проанализируй таблицу» → ждёте → «Теперь создай отчёт» → ждёте → «Теперь презентацию» | 3 итерации, нужно ждать и проверять каждый шаг |
| Один пайплайн | «Проанализируй таблицу → создай отчёт → сделай презентацию» | Cowork выполняет всю цепочку, вы проверяете план один раз |

Когда вы описываете всю цепочку сразу, Cowork видит конечную цель и лучше планирует промежуточные шаги. Он знает, что данные из анализа пойдут в отчёт, а тезисы из отчёта — в презентацию.

**Как читать план пайплайна**

Перед выполнением Cowork покажет план — последовательность шагов. Для пайплайна проверяйте:

1. **Правильный порядок:** сначала данные, потом анализ, потом документы
2. **Связи между шагами:** данные из шага 1 используются в шаге 2, а не читаются заново
3. **Правильные папки:** каждый артефакт сохраняется в указанное место
4. **Все шаги на месте:** если вы попросили 3 артефакта — в плане должно быть 3 шага

Если что-то не так — скорректируйте до одобрения.

**Когда пайплайн — это слишком**

Пайплайн полезен, когда:
- Есть чёткая цепочка: данные → анализ → документ → презентация
- Артефакты связаны общим источником данных
- Задача повторяется регулярно (еженедельно, ежемесячно)

Пайплайн избыточен, когда:
- Задача разовая и простая (одно письмо, один отчёт)
- Шаги независимы друг от друга (не используют общие данные)
- Вам нужно проверить каждый промежуточный результат перед следующим шагом

### Заключение (30 сек)

Пайплайн — это когда Cowork выполняет цепочку задач за вас: от данных в таблице до готовой презентации. Ключевое правило: описывайте весь цикл в одном задании, а не по шагам. В следующем уроке мы применим пайплайн к конкретной задаче — подготовке к рабочей встрече.

## Текстовое сопровождение

### Что такое пайплайн

Пайплайн — цепочка задач, где результат одного шага → вход для следующего:

```
Google Sheets (данные) → Анализ (новый лист) → Google Docs (отчёт) → Google Slides (презентация)
```

**Без Cowork:** 4 инструмента, ручное копирование между ними, 30-60 минут.
**С Cowork:** 1 задание, автоматическая цепочка через gws, 3-5 минут вашего времени.

### Шаблон задания для пайплайна

```
Подготовь пакет документов по [тема]:

1. Открой Google Sheets "[название]".
   Проанализируй: [конкретные вопросы].
   Результат — на листе "[название листа]".

2. На основе анализа создай отчёт в Google Docs
   в папке "[папка]":
   [описание структуры отчёта].

3. Из отчёта создай презентацию в Google Slides
   на [N] слайдов в папке "[папка]":
   [структура слайдов].
   К каждому слайду — speaker notes.
```

### Одно задание vs пошаговые команды

| | Пошагово | Пайплайн |
|--|----------|----------|
| Формулировка | 3 отдельных запроса | 1 комплексное задание |
| Ваше участие | Проверяете каждый шаг | Проверяете план один раз |
| Связность | Вы сами связываете шаги | Cowork видит всю цепочку |
| Время | 15-20 мин (с ожиданием) | 3-5 мин |

### Чеклист плана пайплайна

Перед одобрением плана проверьте:
- [ ] Правильный порядок шагов (данные → анализ → документ → презентация)
- [ ] Данные из шага 1 используются в шаге 2 (не читаются заново)
- [ ] Каждый артефакт сохраняется в указанную папку
- [ ] Все запрошенные шаги присутствуют в плане

## Ключевые выводы

1. **Пайплайн** — цепочка задач, где Cowork автоматически передаёт результат каждого шага на следующий: данные → анализ → отчёт → презентация. Вместо ручного copy-paste между инструментами — одно задание
2. **Описывайте весь пайплайн в одном задании**, а не по шагам. Cowork видит конечную цель и лучше планирует промежуточные шаги. Вы проверяете план один раз, а не три
3. **Пайплайн полезен для регулярных задач** с чёткой цепочкой и общим источником данных. Для разовых простых задач — достаточно одного задания без цепочки
