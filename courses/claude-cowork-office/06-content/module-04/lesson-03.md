## Видео-скрипт

### Вступление (30 сек)

Здравствуйте! Один из самых частых пайплайнов в офисной работе — подготовка к встречам. Собрать данные, подготовить повестку, сформулировать тезисы, сделать шаблон протокола — на это уходит от 30 минут до часа. В этом уроке покажу, как сделать это за 10 минут с Cowork.

### Основная часть

**Пайплайн «до встречи»**

Перед любой рабочей встречей нужно подготовить:
1. Данные — актуальные цифры из таблиц
2. Повестку — что обсуждаем, в каком порядке
3. Тезисы — ключевые моменты по каждому пункту
4. Шаблон протокола — чтобы на встрече фиксировать решения

Cowork может сделать всё это одним заданием:

> «Помоги подготовиться к еженедельной планёрке в понедельник.
> 1. Открой Google Sheets "Задачи_команда" — собери статус по задачам за неделю: выполнено, в работе, просрочено.
> 2. Создай повестку в Google Docs в папке "Планёрки": статус задач (таблица), проблемные задачи (список), вопросы для обсуждения.
> 3. К каждому пункту повестки добавь тезисы — 2-3 предложения, о чём говорить.
> 4. В конце документа добавь шаблон протокола: дата, участники, принятые решения (список), таблица action items (задача | ответственный | срок | статус).»

Результат: один документ в Google Docs с повесткой, тезисами и шаблоном протокола — готов за 3 минуты.

**Пайплайн «после встречи»**

После встречи у вас есть заметки — рукописные или в заметках на телефоне. Cowork превращает их в структурированный протокол:

> «Вот мои заметки с планёрки:
> - обсудили отчёт, Алишер сдаст до пятницы
> - нужно перенести встречу с клиентом на среду
> - бюджет на маркетинг утвердили 15М сум
> - Нилуфар проверит данные по конверсии
> - следующая планёрка в понедельник 10:00
>
> Создай протокол в Google Docs в папке "Протоколы":
> дата и участники, принятые решения (список), таблица action items (задача | ответственный | срок), следующие шаги.»

Cowork берёт сырые заметки и создаёт структурированный документ с таблицей задач. Каждый участник видит свои action items.

**Демонстрация: подготовка за 10 минут вместо часа**

Сравним два подхода к подготовке еженедельной планёрки:

**Без Cowork (45-60 мин):**
1. Открыть таблицу задач, вручную посчитать статусы (10 мин)
2. Открыть Google Docs, написать повестку (15 мин)
3. Вспомнить, что обсуждать по каждому пункту (10 мин)
4. Создать шаблон протокола (10 мин)
5. Отформатировать (5 мин)

**С Cowork (10 мин):**
1. Написать задание с описанием того, что нужно (3 мин)
2. Проверить план и одобрить (2 мин)
3. Проверить готовый документ (5 мин)

Экономия: 35-50 минут каждую неделю. За месяц — 2-3 часа только на подготовке к одной регулярной встрече.

**Совет: шаблон для повторяющихся встреч**

Если у вас есть встреча, которая повторяется каждую неделю, сохраните задание как шаблон. В модуле 5 мы оформим это как skill — готовую инструкцию, которую можно запускать одной командой.

### Заключение (30 сек)

Подготовка к встрече — идеальная задача для пайплайна: данные из таблицы → повестка → тезисы → шаблон протокола. И после встречи: заметки → протокол с action items. Вместо часа — 10 минут. В следующем уроке вы соберёте свой пайплайн на реальных данных.

## Текстовое сопровождение

### Пайплайн «до встречи»

```
Google Sheets (данные) → Google Docs (повестка + тезисы + шаблон протокола)
```

**Что включить:**
- Статус задач из таблицы (выполнено / в работе / просрочено)
- Повестка с пунктами обсуждения
- Тезисы к каждому пункту (2-3 предложения)
- Шаблон протокола: решения + action items

### Пайплайн «после встречи»

```
Заметки (свободный текст) → Google Docs (протокол + action items)
```

**Структура протокола:**
- Дата и участники
- Принятые решения (нумерованный список)
- Таблица action items: задача | ответственный | срок | статус
- Следующие шаги

### Шаблон задания: подготовка к планёрке

```
Помоги подготовиться к [название встречи].

1. Открой Google Sheets "[таблица]" —
   собери статус по задачам за [период].

2. Создай повестку в Google Docs в папке "[папка]":
   - Статус задач (таблица)
   - Проблемные задачи (список)
   - Вопросы для обсуждения

3. К каждому пункту — тезисы (2-3 предложения).

4. Шаблон протокола: дата, участники,
   решения, action items (задача | ответственный | срок).
```
