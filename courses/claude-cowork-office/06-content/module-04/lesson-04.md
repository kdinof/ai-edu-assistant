## Видео-скрипт

### Вступление (30 сек)

Здравствуйте! Это практический урок модуля 4. Вы соберёте полный пайплайн для реальной рабочей задачи: данные из Google Sheets → анализ → отчёт в Google Docs → презентация в Google Slides. Три связанных артефакта, созданных через Cowork и Google Workspace CLI.

### Основная часть

**Задание: полный пайплайн**

Выберите одну из двух задач:

**Вариант A: Пакет документов по результатам**

Подготовьте пакет документов по итогам месяца, квартала или проекта:

1. Выберите таблицу с данными из Google Sheets (продажи, расходы, задачи, метрики)
2. Попросите Cowork проанализировать данные, создать отчёт в Google Docs и сделать презентацию в Google Slides
3. Всё — одним заданием

Пример:
> «Подготовь пакет документов по итогам марта 2026:
> 1. Открой Google Sheets "[ваша таблица]". Проанализируй [конкретные метрики]. Результат — на листе "Анализ".
> 2. Создай отчёт в Google Docs в папке "[ваша папка]": [структура отчёта].
> 3. Создай презентацию в Google Slides на [N] слайдов в папке "[папка]": [структура слайдов]. Speaker notes к каждому слайду.»

**Вариант B: Подготовка к реальной встрече**

Если у вас на этой неделе есть рабочая встреча — подготовьте пакет документов для неё:

1. Соберите данные из Google Sheets (статус задач, метрики проекта)
2. Создайте повестку с тезисами в Google Docs
3. Если для встречи нужна презентация — добавьте её

Пример:
> «Подготовь документы к встрече [название] [дата]:
> 1. Открой Google Sheets "[таблица]" — собери статус по [что нужно].
> 2. Создай повестку в Google Docs: [пункты]. К каждому — тезисы. Шаблон протокола в конце.
> 3. Создай презентацию на [N] слайдов с ключевыми данными для обсуждения.»

**Как выполнять**

1. Выберите вариант A или B
2. Сформулируйте задание, используя фреймворк Что → Откуда → Куда → Формат
3. Опишите весь пайплайн в одном задании (не по шагам!)
4. Проверьте план Cowork перед одобрением
5. Проверьте результат по чеклисту ниже

**Безопасность данных**

Презентации часто показывают внешней аудитории. Перед тем как давать задание Cowork:

- Проверьте, нет ли в исходных данных персональных данных клиентов — они могут попасть в презентацию
- Если отчёт для внутреннего использования — убедитесь, что пометка «конфиденциально» добавлена
- Не включайте в презентацию для клиентов внутренние финансовые данные, зарплаты, маржинальность
- Если сомневаетесь — попросите Cowork использовать агрегированные данные вместо детализации

**Чеклист выполнения**

После завершения проверьте:

- [ ] Задание описывает весь пайплайн целиком (не по шагам)
- [ ] Созданы минимум 3 связанных артефакта (Sheet + Doc + Slides)
- [ ] Все артефакты созданы через Cowork и Google Workspace CLI (не copy-paste)
- [ ] Данные в отчёте и презентации совпадают (взяты из одного источника)
- [ ] Презентация содержит speaker notes
- [ ] В артефактах нет персональных данных и конфиденциальной информации, которая не должна выходить за пределы компании

### Заключение (30 сек)

Поздравляю — вы собрали полноценный рабочий пайплайн. Данные из таблицы → отчёт → презентация, всё через Cowork и Google Workspace CLI. Это четвёртый компонент вашего AI-офиса. В последнем модуле мы превратим такие пайплайны в skills — готовые инструкции, которые запускаются одной командой — и замерим итоговую экономию времени.

## Текстовое сопровождение

### Два варианта задания

**Вариант A — Пакет документов:** данные из Sheets → анализ → отчёт в Docs → презентация в Slides. Подходит для отчётности по итогам периода или проекта.

**Вариант B — Подготовка к встрече:** данные из Sheets → повестка с тезисами в Docs → презентация в Slides. Подходит, если на этой неделе есть реальная встреча.

### Шаблон задания (Вариант A)

```
Подготовь пакет документов по итогам [период]:

1. Google Sheets "[таблица]":
   - Проанализируй [метрики]
   - Результат — на листе "Анализ"

2. Google Docs в папке "[папка]":
   - [структура отчёта]

3. Google Slides на [N] слайдов в папке "[папка]":
   - [структура слайдов]
   - Speaker notes к каждому слайду
```

### Шаблон задания (Вариант B)

```
Подготовь документы к встрече [название] [дата]:

1. Google Sheets "[таблица]":
   - Собери статус по [что нужно]

2. Google Docs в папке "[папка]":
   - Повестка с тезисами к каждому пункту
   - Шаблон протокола в конце

3. Google Slides на [N] слайдов:
   - Ключевые данные для обсуждения
```

### Безопасность данных в презентациях

| Ситуация | Что проверить |
|----------|---------------|
| Презентация для руководства | Внутренние данные допустимы |
| Презентация для клиента | Убрать маржу, зарплаты, внутренние метрики |
| Презентация для партнёра | Только согласованные данные |
| Отчёт для внешней аудитории | Пометка «конфиденциально», если нужно |

### Чеклист

- [ ] Пайплайн описан одним заданием
- [ ] Минимум 3 связанных артефакта (Sheet + Doc + Slides)
- [ ] Всё через Cowork + gws, без copy-paste
- [ ] Данные согласованы между артефактами
- [ ] Speaker notes в презентации
- [ ] Нет утечки конфиденциальных данных

## Ключевые выводы

1. **Полный пайплайн** = минимум 3 связанных артефакта из одного задания: лист анализа в Sheets, отчёт в Docs, презентация в Slides. Все данные согласованы, потому что идут из одного источника
2. **Описывайте весь пайплайн одним заданием.** Чем полнее задание, тем лучше Cowork планирует цепочку — и тем меньше вашего времени уходит на проверку
3. **Безопасность для презентаций** — отдельная проверка: данные из внутренних таблиц могут случайно попасть в слайды для внешней аудитории. Проверяйте перед показом
