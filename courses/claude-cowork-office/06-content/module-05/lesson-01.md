## Видео-скрипт

### Вступление (30 сек)

Здравствуйте! За предыдущие модули вы научились создавать документы, работать с таблицами и собирать пайплайны. Но каждый раз вы писали задание заново. Если у вас есть задача, которую вы делаете каждую неделю — зачем переформулировать задание каждый раз? В этом уроке разберём skills — способ сохранить удачное задание и запускать его одной командой.

### Основная часть

**Что такое skill**

Skill — это сохранённая инструкция для задачи, которую вы делаете регулярно. Вместо того чтобы каждый раз писать «Возьми данные из таблицы, проанализируй, создай отчёт...», вы сохраняете это задание один раз и потом запускаете одной фразой.

Аналогия: skill — это как рецепт. Вы один раз записали ингредиенты и шаги, а потом готовите по нему каждый раз, не вспоминая пропорции заново.

**Откуда брать skills**

Лучшие skills — это задания, которые уже сработали хорошо. За модули 2-4 вы создавали документы, анализировали таблицы, собирали пайплайны. Посмотрите на свои задания:

- Какие задачи вы делаете каждую неделю?
- Какое задание дало хороший результат с первого раза?
- Какое задание вы скопировали бы и использовали снова?

Вот это и есть кандидаты в skills.

**Как создать skill**

Skill в Cowork — это отдельный файл с инструкцией. Создать его можно двумя способами:

**Способ 1 — через интерфейс Cowork:**
1. Откройте Claude Desktop → Settings → Capabilities → Skills
2. Нажмите «Add Skill»
3. Дайте название и вставьте текст инструкции
4. Сохраните

**Способ 2 — попросите Cowork:**
> «Создай skill "Пятничный отчёт" с инструкцией:
> 1. Открой Google Sheets "Задачи_команда"
> 2. Собери статус: выполнено, в работе, просрочено
> 3. Создай отчёт в Google Docs в папке "Отчёты/Еженедельные": таблица статусов, проблемные задачи, 3 ключевых вывода»

Cowork создаст файл skill и сохранит его автоматически.

После создания skill запускается одной фразой:
> «Пятничный отчёт»

Cowork узнаёт задачу по описанию skill и выполняет инструкцию.

**Демонстрация: skill «Пятничный отчёт»**

До skill: каждую пятницу вы тратите 3-5 минут на формулировку задания. Иногда забываете указать папку или формат. Результат каждый раз немного разный.

После skill: одна фраза — и Cowork выполняет по сохранённой инструкции. Результат стабильный каждую неделю.

**Примеры skills для офисных задач**

| Skill | Что делает | Частота |
|-------|-----------|---------|
| Пятничный отчёт | Статус задач → отчёт в Docs | 1 раз/неделю |
| Подготовка к планёрке | Данные из Sheets → повестка + шаблон протокола | 1 раз/неделю |
| Протокол встречи | Заметки → протокол с action items в Docs | 2-3 раза/неделю |
| Анализ расходов | Таблица расходов → сводка + выводы | 1 раз/месяц |
| Деловое письмо | Тема + адресат → письмо в нужном формате | По запросу |

**Правила хороших skills**

1. **Конкретность.** Skill должен содержать все 4 элемента: Что → Откуда → Куда → Формат. Без этого результат будет каждый раз разным
2. **Регулярность.** Создавайте skills для задач, которые делаете минимум раз в неделю. Для разовых задач skill — лишняя работа
3. **Проверенность.** Skill = задание, которое уже дало хороший результат. Не сохраняйте в skill задание, которое ни разу не пробовали

**Антипаттерн: skill для всего**

Не нужно создавать skill для каждого действия. Skill для задачи «ответь на письмо» — это лишнее, потому что каждое письмо уникально. Skill полезен, когда структура задачи повторяется, а меняются только данные.

### Заключение (30 сек)

Skill — это сохранённое задание, которое работает одной командой. Возьмите задачу из модулей 2-4, которая получилась хорошо, и создайте из неё skill через Cowork. В следующем уроке мы разберём, как создавать skills с несколькими шагами — цепочки, которые автоматизируют целый рабочий процесс.

## Текстовое сопровождение

### Что такое skill

Skill — сохранённая инструкция для регулярной задачи. Хранится как отдельный файл в Cowork и запускается одной фразой.

**Без skill:** каждый раз пишете задание заново (3-5 мин), иногда забываете детали, результат нестабильный.
**Со skill:** одна фраза → стабильный результат каждый раз.

### Как создать skill

**Через интерфейс:**
1. Claude Desktop → Settings → Capabilities → Skills → Add Skill
2. Укажите название и инструкцию
3. Сохраните

**Через Cowork:**
> «Создай skill "[название]" с инструкцией: [текст задания]»

**Запуск:**
> «[Название skill]» — Cowork узнаёт задачу и выполняет по инструкции

### Пример skill: Пятничный отчёт

**Название:** Пятничный отчёт

**Инструкция:**
```
Каждую пятницу:
1. Открой Google Sheets "Задачи_команда"
2. Собери статус: выполнено, в работе, просрочено
3. Создай отчёт в Google Docs в папке "Отчёты/Еженедельные":
   - Таблица статусов
   - Проблемные задачи
   - 3 ключевых вывода
4. Формат: таблица + выводы, без воды
```

### Пример skill: Подготовка к планёрке

**Название:** Подготовка к планёрке

**Инструкция:**
```
Каждый понедельник утром:
1. Открой Google Sheets "Задачи_команда" — статус за неделю
2. Создай повестку в Google Docs в папке "Планёрки":
   - Статус задач (таблица)
   - Проблемные задачи (список)
   - Вопросы для обсуждения
3. К каждому пункту — тезисы (2-3 предложения)
4. Шаблон протокола в конце документа
```

### Правила хороших skills

| Правило | Почему |
|---------|--------|
| Конкретность | Без 4 элементов (Что/Откуда/Куда/Формат) результат нестабилен |
| Регулярность | Skill для задачи раз в год — лишняя работа |
| Проверенность | Сохраняйте только задания, которые уже дали хороший результат |

## Ключевые выводы

1. **Skill** — сохранённая инструкция для регулярной задачи. Создаётся через Settings → Skills → Add Skill или через Cowork одной командой. Запускается фразой с названием skill
2. **Лучшие skills** — это задания из модулей 2-4, которые уже дали хороший результат. Не выдумывайте новые — возьмите то, что сработало, и сохраните как skill
3. **Skill полезен для регулярных задач** с повторяющейся структурой (отчёты, подготовка к встречам, протоколы). Для уникальных разовых задач skill создавать не нужно
