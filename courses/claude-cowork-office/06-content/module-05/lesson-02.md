## Видео-скрипт

### Вступление (30 сек)

Здравствуйте! В прошлом уроке мы создали простые skills — шаблоны для отдельных задач. Но многие рабочие процессы состоят из нескольких шагов подряд. Подготовка к концу месяца — это не один отчёт, а цепочка: собрать данные, создать отчёт, сделать презентацию. В этом уроке научимся создавать составные skills и настраивать автоматический запуск по расписанию.

### Основная часть

**Составной skill: цепочка шагов в одной инструкции**

Skill может содержать не только одну задачу, а целую цепочку — как пайплайн из модуля 4, но сохранённый для повторного использования.

Пример: skill «Конец месяца» для бухгалтера:

**Название:** Конец месяца

**Инструкция:**
```
Запускать в последний рабочий день месяца.
Выполни последовательно:
1. Открой Google Sheets "Расходы_[текущий месяц]".
   Создай лист "Сводка" с итогами по категориям.
2. На основе сводки создай отчёт в Google Docs
   в папке "Отчёты/Месячные": таблица расходов,
   сравнение с прошлым месяцем, 3 вывода.
3. Из отчёта создай презентацию в Google Slides
   на 6 слайдов в папке "Презентации":
   метрики, динамика, выводы, план.
   Speaker notes к каждому слайду.
```

Три задачи, которые раньше занимали полдня — теперь одна фраза: «Конец месяца».

Это не отдельная функция — это просто skill с несколькими шагами. Cowork выполняет их последовательно, передавая результат каждого шага в следующий.

**Пример: составной skill «Еженедельная планёрка»**

Этот skill объединяет подготовку к встрече и обработку результатов в одну инструкцию:

**Инструкция:**
```
Подготовка (до встречи):
1. Открой Google Sheets "Задачи_команда" — собери
   статус за неделю.
2. Создай повестку в Google Docs в папке "Планёрки":
   статус задач (таблица), проблемные задачи,
   вопросы для обсуждения.
3. К каждому пункту — тезисы (2-3 предложения).
4. Шаблон протокола в конце: дата, участники,
   решения, action items.
```

Обратите внимание: для подготовки «до» и обработки «после» — лучше создать два отдельных skill, потому что между ними проходит встреча. Второй skill «Протокол встречи» запускается после.

**Как организовать CLAUDE.md вместе со skills**

Skills хранятся отдельно (в Settings → Skills), а CLAUDE.md содержит контекст вашей должности — это разные вещи, и они дополняют друг друга:

- **CLAUDE.md** — кто вы, ваши задачи, формат документов, правила компании
- **Skills** — конкретные инструкции для повторяющихся задач

Cowork использует оба: из CLAUDE.md берёт контекст (должность, формат), а из skill — конкретные шаги. Поэтому CLAUDE.md не нужно дублировать информацию, которая уже есть в skills.

Рекомендуемая структура CLAUDE.md:

```markdown
# Мой AI-ассистент

## Обо мне
[Должность, отдел, компания]

## Правила
[Язык, ограничения, безопасность]

## Формат документов
[Как оформлять отчёты, письма, таблицы]

## Рабочие папки
- Отчёты: Google Drive / Отчёты
- Планёрки: Google Drive / Планёрки
- Презентации: Google Drive / Презентации
```

А конкретные задания (пятничный отчёт, подготовка к планёрке) — в skills.

**Scheduled tasks: автоматический запуск по расписанию**

Cowork позволяет запускать skills автоматически по расписанию. Вы можете настроить:

> «Каждый понедельник в 9:00 запускай skill "Подготовка к планёрке"»

Cowork выполнит задачу и покажет результат для проверки.

**Важное ограничение:** scheduled tasks работают только при включённом компьютере и открытом Claude Desktop. Если Mac в режиме сна или приложение закрыто — задача пропускается и выполнится при следующем пробуждении. Это локальная автоматизация, а не облачный сервис.

Практический совет: если у вас есть задача на утро понедельника — убедитесь, что Claude Desktop запущен до нужного времени. Или запустите skill вручную, когда откроете приложение.

| Задача | Расписание | Когда настроить |
|--------|-----------|----------------|
| Подготовка к планёрке | Пн, 9:00 | Если Mac включён утром |
| Пятничный отчёт | Пт, 16:00 | Если работаете до конца дня |
| Месячная сводка | Последний день, 14:00 | Если Mac включён |

**Антипаттерн: слишком сложный skill**

Не создавайте skill из 10 шагов. Если задача слишком большая — разбейте на 2-3 отдельных skill. Проще запустить два skill по очереди, чем отлаживать один мегасложный.

### Заключение (30 сек)

Составной skill — это пайплайн, сохранённый для повторного использования. А scheduled tasks позволяют запускать skills автоматически по расписанию — но только при включённом Mac. В следующем уроке мы вернёмся к Time Audit и измерим, сколько времени вы действительно сэкономили.

## Текстовое сопровождение

### Составной skill vs простой

| | Простой skill | Составной skill |
|--|--------------|----------------|
| Шаги | 1 задача | 2-4 задачи в цепочке |
| Пример | «Пятничный отчёт» | «Конец месяца» (сводка + отчёт + презентация) |
| Когда использовать | Отдельная повторяющаяся задача | Цепочка задач, которая повторяется вместе |

### Пример составного skill

**Название:** Конец месяца

**Инструкция:**
```
Запускать в последний рабочий день месяца.
1. Google Sheets "Расходы_[месяц]" → лист "Сводка"
2. Сводка → отчёт в Google Docs (папка "Отчёты/Месячные")
3. Отчёт → презентация в Google Slides на 6 слайдов
   (папка "Презентации") + speaker notes
```

### CLAUDE.md + Skills: разделение ответственности

| CLAUDE.md | Skills |
|-----------|--------|
| Кто вы (должность, компания) | Конкретные инструкции задач |
| Правила и формат документов | Шаги выполнения |
| Рабочие папки и инструменты | Источники данных и результаты |

CLAUDE.md → контекст. Skills → действия.

### Scheduled tasks

**Возможности:** автоматический запуск skill по расписанию (ежедневно, еженедельно, ежемесячно).

**Ограничение:** работает только при включённом Mac и открытом Claude Desktop. Если компьютер спит — задача пропускается.

**Рекомендация:** для утренних задач убедитесь, что Claude Desktop запущен. Или запускайте skill вручную при открытии приложения.

## Ключевые выводы

1. **Составной skill** — пайплайн из нескольких шагов, сохранённый для повторного использования. Создаётся так же, как обычный skill, но с несколькими шагами в инструкции
2. **CLAUDE.md и skills дополняют друг друга:** CLAUDE.md хранит контекст (должность, правила, формат), skills — конкретные инструкции. Не дублируйте информацию между ними
3. **Scheduled tasks** запускают skills автоматически по расписанию, но только при включённом Mac и открытом Claude Desktop — это локальная автоматизация, не облачный сервис
