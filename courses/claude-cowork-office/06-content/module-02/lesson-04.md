## Видео-скрипт

### Вступление (30 сек)

Теория позади — пора применить всё на своих данных. В этом уроке вы создадите три реальных рабочих документа через Cowork и Google Docs. Не учебных примеров, а настоящих документов, которые вы сможете использовать завтра на работе. Готовы? Давайте начнём.

### Основная часть

**Подготовка (1 мин)**

Прежде чем начать, подготовьте исходные данные. Вам понадобятся:
- Любая рабочая таблица в Google Sheets (продажи, расходы, трекер задач — что угодно)
- Заметки с последней встречи (в Google Docs или текстовом файле)
- Информация для делового письма (кому, о чём, зачем)

Если у вас нет таблицы в Google Sheets — создайте простую: 10-15 строк с данными по вашим задачам за последнюю неделю. Этого достаточно.

**Документ 1: Отчёт (4 мин)**

Откройте Claude Cowork и создайте отчёт на основе своих данных. Используйте фреймворк из урока 2.1.

Шаблон задания:
> «Открой Google Sheets "[название вашей таблицы]". Проанализируй данные за [период]. Создай отчёт в Google Docs в папке "[ваша папка]":
> - Заголовок: "[название отчёта]"
> - Таблица ключевых метрик: [перечислите 3-5 метрик]
> - Сравнение с предыдущим периодом
> - 3 ключевых вывода
> - Рекомендации»

Проверьте план Cowork перед одобрением. После создания — откройте документ в Google Docs и убедитесь, что он выглядит так, как нужно. Если нужны правки — примените технику итерации из урока 2.3.

**Документ 2: Деловое письмо (3 мин)**

Создайте письмо, которое вам реально нужно отправить: клиенту, партнёру, коллеге из другого отдела, руководителю.

Шаблон задания:
> «Создай деловое письмо в Google Docs в папке "[ваша папка]".
> Адресат: [кому].
> Тема: [о чём].
> Тон: [профессиональный / дружелюбный / формальный].
> Структура: [перечислите секции].
> Подпись: [ваши данные]»

Важно: не забудьте указать тон письма. «Профессиональный, на "вы"» — это не то же самое, что «дружелюбный, полуформальный». Cowork подстроит стиль под ваше указание.

**Документ 3: Протокол или summary (3 мин)**

Выберите один из вариантов:

Вариант А — протокол встречи:
> «Прочитай [ваши заметки] в Google Docs. Создай протокол в новом Google Doc в папке "[папка]":
> - Дата, участники
> - Обсуждённые вопросы
> - Принятые решения (с ответственными)
> - Таблица задач: задача | ответственный | срок
> - Следующие шаги»

Вариант Б — summary длинного документа:
> «Прочитай документ "[название]" в Google Docs. Создай summary в новом Google Doc:
> - Краткое содержание (5-7 предложений)
> - Ключевые пункты (список)
> - Вопросы для уточнения»

**Безопасность данных**

Перед тем как загружать реальные рабочие документы, проверьте:
- Нет ли в них персональных данных клиентов (ФИО + телефон + адрес)?
- Нет ли паролей, ключей доступа, номеров банковских карт?
- Нет ли конфиденциальных цифр, защищённых NDA?

Если есть — анонимизируйте перед использованием. Замените имена на «Клиент 1», «Клиент 2», уберите точные суммы, если они под NDA. Cowork работает в защищённой среде, но привычка проверять данные перед отправкой в AI — это хорошая практика для любого инструмента.

### Заключение (30 сек)

Вы создали три реальных документа прямо в Google Docs через Cowork — без единого copy-paste. Посмотрите на результат: отчёт с данными из таблицы, деловое письмо с правильным тоном и протокол с задачами. Всё это заняло около 10 минут. Сколько бы вы потратили, делая это вручную? В следующем модуле мы перейдём к таблицам и данным — научим Cowork анализировать Google Sheets и создавать таблицы с формулами.

## Текстовое сопровождение

### Чеклист подготовки

Перед началом убедитесь, что у вас есть:

- [ ] Рабочая таблица в Google Sheets (любые данные: продажи, задачи, расходы)
- [ ] Заметки с последней встречи (Google Docs или текстовый файл)
- [ ] Информация для делового письма (адресат, тема, цель)
- [ ] Папки в Google Drive для сохранения документов

### Задание 1: Отчёт

**Шаблон:**
```
Открой Google Sheets "[название таблицы]".
Проанализируй данные за [период].
Создай отчёт в Google Docs в папке "[папка]":
- Заголовок: "[название отчёта]"
- Таблица метрик: [3-5 метрик]
- Сравнение с предыдущим периодом
- 3 ключевых вывода
- Рекомендации
```

**Критерии:**
- Документ создан в Google Docs (не в чате)
- Данные взяты из Google Sheets через CLI (не скопированы вручную)
- Есть таблица, выводы и рекомендации

### Задание 2: Деловое письмо

**Шаблон:**
```
Создай деловое письмо в Google Docs в папке "[папка]".
Адресат: [кому].
Тема: [о чём].
Тон: [какой].
Структура: [секции].
Подпись: [данные].
```

**Критерии:**
- Правильный тон (соответствует указанному)
- Чёткая структура (приветствие → суть → условия → призыв)
- Документ в Google Docs

### Задание 3: Протокол или summary

**Вариант А — протокол:**
```
Прочитай [заметки] в Google Docs.
Создай протокол в новом Google Doc в папке "[папка]":
- Дата, участники
- Обсуждённые вопросы
- Решения (с ответственными)
- Таблица задач: задача | ответственный | срок
- Следующие шаги
```

**Вариант Б — summary:**
```
Прочитай документ "[название]" в Google Docs.
Создай summary в новом Google Doc:
- Краткое содержание (5-7 предложений)
- Ключевые пункты (список)
- Вопросы для уточнения
```

**Критерии:**
- Информация извлечена из исходного документа (не придумана)
- Есть структура с задачами/пунктами
- Документ в Google Docs

### Чеклист безопасности данных

Перед загрузкой документов проверьте:

| Проверка | Действие |
|----------|---------|
| Персональные данные клиентов (ФИО + контакты) | Заменить на «Клиент 1», «Клиент 2» |
| Пароли, ключи доступа | Удалить перед загрузкой |
| Номера банковских карт | Никогда не загружать |
| Суммы под NDA | Заменить на приблизительные или относительные |
| Внутренние стратегические документы | Оценить риск перед загрузкой |

### Финальный чеклист модуля 2

- [ ] Документ 1 (отчёт) создан в Google Docs через Cowork
- [ ] Документ 2 (письмо) создан в Google Docs через Cowork
- [ ] Документ 3 (протокол/summary) создан в Google Docs через Cowork
- [ ] Все документы созданы через Google Workspace CLI, без copy-paste
- [ ] Данные проверены на безопасность перед загрузкой

## Ключевые выводы

1. **Практика на своих данных** — главный способ закрепить навык. Учебные примеры показывают принцип, но только реальные рабочие документы доказывают, что Cowork экономит ваше время
2. **Три типа документов покрывают 80% офисной работы:** отчёты из данных, деловые письма и протоколы/summary. Освоив их, вы сможете адаптировать фреймворк для любого другого документа
3. **Безопасность данных — ваша ответственность.** Перед загрузкой рабочих документов в Cowork проверяйте: нет ли персональных данных, паролей, информации под NDA. Анонимизируйте при необходимости
