## Видео-скрипт

### Вступление (30 сек)

В прошлом уроке мы разобрали фреймворк постановки задачи: Что → Откуда → Куда → Формат. Теперь давайте посмотрим, как это работает на практике. Мы создадим три типа документов, которые делает почти каждый офисный работник: еженедельный отчёт, деловое письмо и протокол встречи. И всё это — прямо в Google Docs через Cowork.

### Основная часть

**Сценарий 1: Еженедельный отчёт по продажам (3 мин)**

Представьте: каждую пятницу вы готовите отчёт для руководителя. Обычно это занимает час-полтора: открыть таблицу, выгрузить данные, посчитать метрики, написать выводы, оформить в документ.

Давайте поручим это Cowork. Переходим в Cowork и пишем задание:

> «Открой Google Sheets "Продажи_март_2026". Проанализируй данные за последнюю неделю (4-10 марта). Создай еженедельный отчёт в Google Docs в папке "Отчёты/Март":
> - Заголовок: "Еженедельный отчёт по продажам: 4-10 марта 2026"
> - Таблица ключевых метрик: выручка, количество сделок, средний чек, конверсия
> - Сравнение с предыдущей неделей (абсолютные числа и проценты)
> - Топ-3 продукта по выручке
> - 3 ключевых вывода для руководителя
> - Рекомендации на следующую неделю»

Cowork показывает план: сначала откроет таблицу, найдёт нужные строки, посчитает метрики, затем создаст документ. Проверяем — всё верно, одобряем.

Через пару минут — готовый отчёт в Google Docs. С таблицами, сравнениями, выводами. Вам осталось только просмотреть и отправить руководителю.

Обратите внимание: мы не копировали данные из таблицы в чат. Cowork сам прочитал Google Sheets через Google Workspace CLI и создал документ. Никакого copy-paste.

**Сценарий 2: Деловое письмо (2 мин)**

Теперь создадим деловое письмо. Допустим, нужно написать партнёру предложение о сотрудничестве.

> «Создай деловое письмо в Google Docs в папке "Письма". Адресат: директор по развитию компании Korzinka. Тема: предложение по обучению сотрудников работе с AI-инструментами.
> Структура:
> - Приветствие (формальное, на "вы")
> - Кто мы: OSNOVA — образовательная платформа в Узбекистане
> - Суть предложения: корпоративное обучение AI-инструментам для офисных сотрудников (2 абзаца)
> - Конкретные условия: формат, длительность, стоимость на группу 15 человек
> - Призыв к встрече для обсуждения деталей
> - Подпись: Кирилл Гайнутдинов, руководитель направления обучения»

Cowork создаёт письмо с правильной структурой и тоном. Вы проверяете, вносите правки при необходимости — и письмо готово.

**Сценарий 3: Протокол встречи (2 мин)**

Третий типичный документ — протокол встречи. После планёрки у вас есть заметки, и нужно превратить их в структурированный протокол с задачами.

> «Прочитай документ "Заметки_планёрка_10марта" в Google Docs. Создай протокол встречи в новом Google Doc в папке "Протоколы":
> - Заголовок: "Протокол еженедельной планёрки — 10 марта 2026"
> - Дата, время, участники (извлеки из заметок)
> - Обсуждённые вопросы (пронумерованный список)
> - Принятые решения (список с ответственными)
> - Таблица задач: задача | ответственный | срок | статус
> - Следующая встреча: дата и повестка»

Здесь Cowork делает то, что отнимает больше всего времени: извлекает из сырых заметок структурированную информацию. Вам не нужно перечитывать заметки и вручную выписывать задачи — Cowork сделает это за вас.

**Сценарий 4: Анализ входящего документа**

Ещё один частый случай — вам прислали длинный документ (договор, отчёт, бриф), и нужно быстро понять суть.

> «Прочитай документ "Бриф_проекта_Uzum" в Google Docs. Создай summary в новом Google Doc в папке "Аналитика":
> - Краткое содержание (5-7 предложений)
> - Ключевые требования (список)
> - Сроки и бюджет
> - Риски и вопросы, которые нужно уточнить»

Вместо того чтобы читать 15-страничный документ целиком, вы получаете структурированную выжимку за минуту.

**Антипаттерн: «улучши текст»**

Самая частая ошибка при работе с документами — просить Cowork «улучшить текст» без контекста. «Улучши» — это не задание. Улучшить как? Сделать короче? Формальнее? Добавить данные? Убрать воду?

Вместо «улучши этот отчёт» скажите конкретно:
- «Сократи отчёт до 1 страницы, оставь только ключевые метрики и выводы»
- «Добавь сравнение с прошлым месяцем в каждую секцию»
- «Перепиши выводы в формате: проблема → причина → рекомендация»

Чем конкретнее задание, тем точнее результат.

### Заключение (30 сек)

Мы создали четыре типа документов: отчёт из данных, деловое письмо, протокол встречи и summary входящего документа. Все — прямо в Google Docs, без copy-paste. Обратите внимание на паттерн: каждый раз мы указывали Что, Откуда, Куда и Формат. В следующем уроке разберём, как доработать результат, когда Cowork сделал не совсем то, что вы хотели.

## Текстовое сопровождение

### Четыре типа документов через Cowork

#### 1. Еженедельный отчёт

**Задание:**
```
Открой Google Sheets "Продажи_март_2026".
Проанализируй данные за последнюю неделю (4-10 марта).
Создай еженедельный отчёт в Google Docs в папке "Отчёты/Март":
- Таблица метрик: выручка, сделки, средний чек, конверсия
- Сравнение с предыдущей неделей (числа и проценты)
- Топ-3 продукта по выручке
- 3 ключевых вывода
- Рекомендации на следующую неделю
```

**Что происходит за кулисами:**
1. Cowork выполняет `gws sheets get` — читает данные из таблицы
2. Анализирует числа, считает метрики и сравнения
3. Выполняет `gws docs create` — создаёт документ с форматированием
4. Сохраняет в указанную папку через `gws drive`

#### 2. Деловое письмо

**Задание:**
```
Создай деловое письмо в Google Docs в папке "Письма".
Адресат: директор по развитию компании Korzinka.
Тема: предложение по обучению сотрудников AI-инструментам.
Тон: профессиональный, на "вы".
Структура: приветствие, кто мы, суть предложения,
условия (формат, длительность, стоимость), призыв к встрече, подпись.
```

#### 3. Протокол встречи

**Задание:**
```
Прочитай "Заметки_планёрка_10марта" в Google Docs.
Создай протокол в новом Google Doc в папке "Протоколы":
- Дата, время, участники
- Обсуждённые вопросы (список)
- Принятые решения (с ответственными)
- Таблица задач: задача | ответственный | срок | статус
- Следующая встреча: дата и повестка
```

#### 4. Summary входящего документа

**Задание:**
```
Прочитай "Бриф_проекта_Uzum" в Google Docs.
Создай summary в новом Google Doc в папке "Аналитика":
- Краткое содержание (5-7 предложений)
- Ключевые требования (список)
- Сроки и бюджет
- Риски и вопросы для уточнения
```

### Антипаттерн: расплывчатые задания

| Плохо (расплывчато) | Хорошо (конкретно) |
|----------------------|-------------------|
| «Улучши этот отчёт» | «Сократи до 1 страницы, оставь метрики и выводы» |
| «Перепиши красивее» | «Добавь таблицу сравнения с прошлым месяцем» |
| «Сделай лучше» | «Перепиши выводы: проблема → причина → рекомендация» |
| «Напиши письмо» | «Создай письмо партнёру: тон, структура, адресат — [детали]» |

### Общий паттерн работы с документами

```
1. Поставить задание (Что → Откуда → Куда → Формат)
2. Проверить план Cowork
3. Одобрить выполнение
4. Просмотреть результат в Google Docs
5. При необходимости — дать конкретную правку (урок 2.3)
```

## Ключевые выводы

1. **Cowork создаёт документы напрямую в Google Docs** через Google Workspace CLI — без copy-paste. Отчёты, письма, протоколы, summary — любой тип офисного документа
2. **Каждое задание строится по фреймворку** «Что → Откуда → Куда → Формат». Чем конкретнее задание, тем меньше итераций потребуется для получения нужного результата
3. **«Улучши текст» — не задание.** Вместо расплывчатых запросов указывайте конкретно: что изменить, какую структуру использовать, какие данные добавить или убрать
