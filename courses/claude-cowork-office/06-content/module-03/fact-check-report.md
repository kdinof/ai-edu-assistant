## Урок 3.1: Claude + Google Sheets: что возможно

### Заявление 1: «Cowork может открыть любую таблицу из Google Drive, прочитать содержимое листов»
**Вердикт: CONFIRMED**
Доступны инструменты `readSpreadsheet`, `getSpreadsheetInfo`, `listGoogleSheets`, `searchDrive`. Чтение данных полностью поддерживается, включая выбор конкретного листа и диапазона (A1-нотация). Параметр `maxRows` позволяет ограничить объём.

### Заявление 2: «Cowork может создать новый лист с результатом прямо в той же таблице»
**Вердикт: CONFIRMED**
Инструмент `addSpreadsheetSheet` создаёт новый лист/вкладку в существующей таблице. Затем `writeSpreadsheet` записывает данные на этот лист.

### Заявление 3: «Создавать таблицы с нуля — Cowork создаёт новую таблицу»
**Вердикт: CONFIRMED**
Инструмент `createSpreadsheet` создаёт новый Google Sheet с заданным названием и опциональными начальными данными (`initialData`). Поддерживается указание папки (`parentFolderId`).

### Заявление 4: «Cowork сам напишет формулу SUMIF или VLOOKUP и вставит её в нужную ячейку»
**Вердикт: CONFIRMED**
`writeSpreadsheet` поддерживает `valueInputOption: "USER_ENTERED"` (значение по умолчанию). При этом строки вида `=SUM(A1:A10)`, `=SUMIF(...)`, `=VLOOKUP(...)` интерпретируются как формулы — аналогично ручному вводу в Google Sheets UI.

### Заявление 5: «Cowork может дописывать строки в существующую таблицу»
**Вердикт: CONFIRMED**
Инструмент `appendSpreadsheetRows` добавляет строки в конец листа.

### Заявление 6: «Google Sheets MCP пока не поддерживает создание графиков»
**Вердикт: CONFIRMED**
В доступном наборе MCP-инструментов Google Workspace нет ни одного инструмента для создания или управления графиками/диаграммами. Хотя Google Sheets API технически поддерживает создание графиков через `batchUpdate`, данный MCP-сервер эту функциональность не реализует.

### Заявление 7: «Cowork может создать файл Excel с графиками через code execution»
**Вердикт: UNCONFIRMED**
Claude Code может генерировать Python-скрипты для создания Excel-файлов с графиками (через openpyxl/xlsxwriter), но это зависит от наличия Python и библиотек в среде выполнения. Утверждение технически возможно, но не является стандартной функцией MCP и может не работать в контексте «Claude Cowork» у обычного пользователя.

### Заявление 8: «Apps Script — не территория MCP»
**Вердикт: CONFIRMED**
В MCP-инструментах нет ничего связанного с Apps Script. MCP работает через Google Sheets API (REST), а не через Apps Script.

### Заявление 9: «Очень большие таблицы (10 000+ строк) — MCP передаёт данные текстом, и на больших объёмах это может быть медленно»
**Вердикт: CONFIRMED**
`readSpreadsheet` возвращает данные как текст (массивы значений). При больших объёмах это увеличивает размер контекстного окна и время обработки. Параметр `maxRows` существует именно для ограничения объёма. Порог 10 000+ строк — разумная оценка, хотя точная граница зависит от ширины таблицы и размера данных.

### Заявление 10: «Тонкое форматирование (цвета, заливка, объединение ячеек) пока ограничено»
**Вердикт: CONFIRMED**
В доступных MCP-инструментах нет инструментов для форматирования ячеек Google Sheets (цвета, шрифты, объединение, границы). Google Sheets API это поддерживает через `batchUpdate`, но MCP-сервер не реализует эти возможности. Утверждение «ограничено» — корректно, точнее было бы сказать «не поддерживается».

## Урок 3.2: Анализ существующих таблиц

### Заявление 11: «Cowork может открыть таблицу через MCP, прочитать данные, сгруппировать, посчитать суммы»
**Вердикт: CONFIRMED**
Чтение — через `readSpreadsheet`. Группировка и подсчёт сумм выполняются самим Claude (в контексте LLM), а результаты записываются обратно через `writeSpreadsheet` или `addSpreadsheetSheet` + `writeSpreadsheet`.

### Заявление 12: «Cowork может работать с несколькими листами одной таблицы»
**Вердикт: CONFIRMED**
`readSpreadsheet` принимает диапазон с указанием листа (например, `"Sheet1!A1:B10"`). `getSpreadsheetInfo` показывает все листы. Можно читать разные листы последовательно.

### Заявление 13: «Вместо часа ручной работы с фильтрами и SUMIF — 2 минуты»
**Вердикт: UNCONFIRMED**
Конкретное время выполнения зависит от размера данных, сложности задачи и скорости MCP-вызовов. «2 минуты» — маркетинговая оценка, которую нельзя верифицировать как факт.

## Урок 3.3: Создание таблиц с формулами

### Заявление 14: «Cowork сам выберет подходящую функцию — SUM, AVERAGE, SUMIF, SUMIFS, VLOOKUP, INDEX/MATCH»
**Вердикт: CONFIRMED**
Все перечисленные функции — стандартные функции Google Sheets. При записи через `writeSpreadsheet` с `USER_ENTERED` они будут корректно интерпретированы как формулы. Claude как LLM знает синтаксис этих функций.

### Заявление 15: «SUM, SUMIF, VLOOKUP, INDEX/MATCH работают в Google Sheets»
**Вердикт: CONFIRMED**
Все перечисленные функции — стандартные встроенные функции Google Sheets: `SUM`, `AVERAGE`, `AVERAGEIF`, `SUMIF`, `SUMIFS`, `VLOOKUP`, `INDEX`, `MATCH`.

### Заявление 16: «Создай трекер задач с выпадающим списком (Новая/В работе/На проверке/Готово)»
**Вердикт: INCORRECT**
В доступных MCP-инструментах нет инструмента для создания data validation / выпадающих списков в Google Sheets. Cowork может записать текстовые значения статусов, но не может создать выпадающий список (dropdown) через MCP. Пользователю придётся добавить data validation вручную. Урок создаёт ложное ожидание, что Cowork сделает выпадающий список автоматически.

### Заявление 17: «Cowork может использовать данные из Google Sheets для создания документов в Google Docs»
**Вердикт: CONFIRMED**
MCP включает инструменты как для Sheets (`readSpreadsheet`), так и для Docs (`createDocument`, `appendToGoogleDoc`, и др.). Claude может прочитать данные из Sheets и записать их в Docs в рамках одной сессии.

### Заявление 18: «Процент от общего — деление на SUM с фиксацией ячейки ($)»
**Вердикт: CONFIRMED**
Абсолютные ссылки с `$` (например, `=B2/$B$10`) — стандартная функциональность Google Sheets. При записи через `USER_ENTERED` корректно интерпретируются.

## Урок 3.4: Практика: анализируем свои данные

### Заявление 19: «Cowork может анализировать реальную таблицу через MCP и создать результат на новом листе»
**Вердикт: CONFIRMED**
Подтверждается комбинацией `readSpreadsheet` → обработка Claude → `addSpreadsheetSheet` + `writeSpreadsheet`.

### Заявление 20: «Данные из Sheets можно использовать для создания документа в Google Docs — первый мини-пайплайн»
**Вердикт: CONFIRMED**
Технически возможно в рамках одной сессии Claude через последовательные MCP-вызовы: чтение из Sheets → создание/запись в Docs.

### Заявление 21: «Все три артефакта (лист анализа, сводка, документ) созданы через Cowork, без ручного copy-paste»
**Вердикт: CONFIRMED**
Все необходимые инструменты доступны: `readSpreadsheet`, `addSpreadsheetSheet`, `writeSpreadsheet`, `createDocument`/`appendToGoogleDoc`.
