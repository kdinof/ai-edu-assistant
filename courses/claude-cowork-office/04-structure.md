# Claude Cowork для офисных работников

## Паспорт курса

| Параметр | Значение |
|----------|----------|
| Длительность | ~2.5 часа |
| Формат | Видео-скринкасты (5-15 мин/урок) + текстовый дубль |
| Уровень | Начинающий |
| Пререквизиты | Mac с Claude Desktop, подписка Claude Pro/Max, Google Workspace аккаунт |
| Основной инструмент | Claude Cowork (Desktop) |
| Вспомогательный | Google Workspace MCP Server |
| Сквозной проект | Персональный AI-офис: Claude Cowork + Google Workspace MCP + библиотека skills + Time Audit |
| Финальный артефакт | Работающий AI-офис с подключённым Google Workspace, библиотекой skills и измеренной экономией времени |

---

## Сквозной проект

Студент пошагово строит **персональный AI-офис** — не набор разрозненных промптов, а систему, где Claude Cowork подключён к рабочим инструментам и выполняет задачи автономно.

| Модуль | Вклад в сквозной проект |
|--------|------------------------|
| 1 | Настроенная среда: Cowork + Google Workspace MCP + CLAUDE.md + Time Audit "до" |
| 2 | Claude создаёт и редактирует Google Docs: отчёты, письма, протоколы |
| 3 | Claude анализирует и создаёт Google Sheets: таблицы с формулами, сводные |
| 4 | Полный пайплайн через MCP: Sheets → анализ → Docs отчёт → презентация |
| 5 | Библиотека skills + автоматизированные workflow + Time Audit "после" |

---

## Структура курса

### Модуль 1: Настройка AI-офиса (30 мин)

**Цель модуля:** Студент настраивает Claude Cowork, подключает Google Workspace через MCP и понимает разницу между чат-ботом и агентом.

**Вклад в сквозной проект:** Работающая среда с MCP + CLAUDE.md с контекстом должности + Time Audit "до"

**Урок 1.1 — Чат-бот vs агент: почему Cowork — это другой подход** (5 мин)
- Чат-бот: промпт → текст → копируй. Агент: задача → план → готовый результат в рабочих инструментах
- Демонстрация: одна и та же задача в ChatGPT (copy-paste) vs в Cowork (результат сразу в Google Docs)
- Что такое MCP: протокол, который позволяет Claude работать с вашими инструментами напрямую
- Главная ошибка: использовать Cowork как чат-бот для генерации текста

**Урок 1.2 — Установка и настройка Cowork + MCP** (10 мин)
- Установка Claude Desktop на Mac, выбор подписки (Pro vs Max)
- Подключение Google Workspace MCP Server: пошаговая инструкция
- Проверка подключения: Claude видит ваши Google Docs, Sheets, Drive
- **Безопасность данных:** Что видит Claude через MCP, как работают permissions, какие данные безопасно подключать, какие — нет (персональные данные, финансовая отчётность, NDA-документы)

**Урок 1.3 — CLAUDE.md: контекст вашей должности** (5 мин)
- Что такое CLAUDE.md — файл-инструкция, который делает Claude вашим персональным ассистентом
- Что включить: должность, типичные задачи, формат документов, правила компании, ограничения
- Демонстрация: CLAUDE.md для бухгалтера vs маркетолога — как контекст меняет качество ответов
- Антипаттерн: пустой CLAUDE.md = Claude не знает кто вы и что вам нужно

**Урок 1.4 — Практика: настраиваем свой AI-офис** (10 мин)
- **Задание на своих данных:** Установить Cowork, подключить Google Workspace MCP, создать CLAUDE.md для своей должности
- Замер Time Audit "до": выбрать 3-5 повторяющихся рабочих задач, записать сколько времени каждая занимает сейчас
- Чеклист: Cowork установлен ✓ MCP подключён ✓ Google Docs/Sheets видны ✓ CLAUDE.md написан ✓ Time Audit "до" заполнен ✓
- **Безопасность данных:** Проверить что через MCP подключены только рабочие (не личные) аккаунты

### Модуль 2: Документы и отчёты (30 мин)

**Цель модуля:** Студент умеет ставить Cowork задачи по созданию и редактированию документов прямо в Google Docs через MCP.

**Вклад в сквозной проект:** Skills для работы с документами, первые готовые docs в Google Workspace

**Урок 2.1 — Как ставить задачу агенту (не промпт, а задание)** (7 мин)
- Разница: промпт ("напиши отчёт") vs задание ("создай еженедельный отчёт по продажам в Google Docs на основе данных за эту неделю, формат: таблица + 3 ключевых вывода")
- Фреймворк постановки задачи: Что сделать → Откуда взять данные → Куда положить результат → В каком формате
- Демонстрация: Cowork создаёт Google Doc с отчётом — план → одобрение → готовый документ
- Как читать план Cowork: на что обращать внимание перед одобрением

**Урок 2.2 — Отчёты, письма, протоколы через Google Docs** (8 мин)
- Создание еженедельного отчёта: Cowork берёт данные из файлов/Sheets → формирует отчёт → сохраняет в Google Docs
- Деловые письма: задать тон, структуру, адресата — получить готовое письмо в Docs
- Протокол встречи: загрузить заметки → Cowork формирует протокол с action items
- Анализ входящего PDF: Cowork читает документ и создаёт summary в Google Docs
- Антипаттерн: просить "улучши текст" вместо конкретного задания с контекстом

**Урок 2.3 — Итерация: как доработать результат** (5 мин)
- Cowork сделал не то — как скорректировать (не переписывать промпт, а дать feedback)
- Chain of corrections: "сделай таблицу компактнее", "добавь сравнение с прошлым месяцем"
- Когда проще доработать руками, а когда — дать задачу Cowork заново
- Демонстрация: три итерации одного отчёта — от черновика до готового документа

**Урок 2.4 — Практика: создаём 3 рабочих документа** (10 мин)
- **Задание на своих данных:** Создать через Cowork + Google Docs MCP три документа из своей работы:
  1. Отчёт (еженедельный/месячный — по реальным данным)
  2. Деловое письмо (клиенту, партнёру или коллеге)
  3. Протокол или summary (по реальным заметкам или документу)
- Критерий: документы созданы прямо в Google Docs через MCP, не через copy-paste
- **Безопасность данных:** Перед загрузкой проверить — нет ли в документах персональных данных клиентов, паролей, конфиденциальных цифр. Как анонимизировать при необходимости

### Модуль 3: Таблицы и данные (30 мин)

**Цель модуля:** Студент умеет ставить Cowork задачи по анализу данных и созданию таблиц в Google Sheets через MCP.

**Вклад в сквозной проект:** Skills для работы с таблицами, интеграция с документами из модуля 2

**Урок 3.1 — Claude + Google Sheets: что возможно** (5 мин)
- Cowork через MCP читает существующие Sheets, анализирует данные, создаёт новые таблицы
- Возможности: формулы, сводные таблицы, фильтрация, вычисления, форматирование
- Демонстрация: Cowork анализирует таблицу продаж → находит тренды → создаёт summary
- Ограничения: что Claude делает хорошо (структура, формулы, анализ), что плохо (сложные графики, макросы)

**Урок 3.2 — Анализ существующих таблиц** (8 мин)
- Постановка задачи: "проанализируй таблицу расходов за Q1, найди топ-5 категорий, посчитай динамику по месяцам"
- Cowork читает Google Sheet → создаёт новый лист с анализом
- Как формулировать запросы к данным: конкретные вопросы вместо "проанализируй"
- Демонстрация: из сырой таблицы на 500 строк → 5 ключевых выводов + сводная таблица

**Урок 3.3 — Создание таблиц с формулами** (7 мин)
- Задание: "создай бюджет проекта на Q2 в Google Sheets с формулами: сумма, среднее, отклонение от плана"
- Cowork создаёт Sheet с правильными формулами и форматированием
- Шаблоны: трекер задач, бюджет, план мероприятий, табель рабочего времени
- Антипаттерн: просить "сделай таблицу" без указания структуры, колонок и формул

**Урок 3.4 — Практика: анализируем свои данные** (10 мин)
- **Задание на своих данных:** Выбрать одну реальную рабочую таблицу из Google Sheets и:
  1. Попросить Cowork проанализировать данные и сделать выводы
  2. Создать новый Sheet с сводной таблицей или дашбордом
  3. Связать результат с документом из модуля 2 (например: данные из Sheets → упоминание в отчёте)
- Критерий: Cowork работает с Google Sheets через MCP, результат — готовый Sheet с формулами
- **Безопасность данных:** Финансовые данные компании — можно ли загружать? Правило: агрегированные данные — да, детализация по клиентам с именами — анонимизировать

### Модуль 4: Презентации и пайплайны (30 мин)

**Цель модуля:** Студент умеет создавать презентации и собирать пайплайны из нескольких шагов, где Cowork оркестрирует работу через MCP.

**Вклад в сквозной проект:** Полный пайплайн данные → анализ → отчёт → презентация через MCP

**Урок 4.1 — Презентации через Claude** (7 мин)
- Два подхода: Google Slides через MCP (структура + контент) и PPTX через code execution (больше контроля над дизайном)
- Постановка задачи: "создай презентацию на 10 слайдов по результатам квартального отчёта для руководства"
- Демонстрация: Cowork берёт данные из Sheets + текст из Docs → создаёт презентацию
- Что Claude делает хорошо: структура, тезисы, таблицы, speaker notes. Что нет: сложный дизайн, анимации

**Урок 4.2 — Пайплайн: данные → отчёт → презентация** (8 мин)
- Что такое пайплайн: цепочка задач, где результат одного шага — вход для следующего
- Демонстрация полного цикла через MCP:
  1. Google Sheets: данные по продажам за месяц
  2. Cowork анализирует → создаёт сводку
  3. Cowork формирует отчёт в Google Docs с выводами
  4. Cowork создаёт презентацию из отчёта
- Одна задача — четыре артефакта. Всё через MCP, без copy-paste
- Как ставить пайплайн-задачу: описать весь цикл в одном задании, а не по шагам

**Урок 4.3 — Подготовка к встречам** (5 мин)
- Пайплайн для встречи: собрать данные → подготовить повестку → тезисы для обсуждения → шаблон протокола
- После встречи: заметки → протокол с action items → рассылка участникам
- Демонстрация: 15-минутная подготовка к еженедельной планёрке вместо часа

**Урок 4.4 — Практика: собираем свой пайплайн** (10 мин)
- **Задание на своих данных:** Собрать полный пайплайн для реальной рабочей задачи:
  1. Выбрать данные из своего Google Sheets
  2. Попросить Cowork проанализировать → создать отчёт в Docs → создать презентацию
  3. Весь цикл — через MCP, одним заданием
- Альтернативное задание: подготовить пакет документов для реальной предстоящей встречи
- Критерий: минимум 3 связанных артефакта (Sheet + Doc + Slides/PPTX), созданных Cowork через MCP
- **Безопасность данных:** Презентации часто показывают внешней аудитории — убедиться, что в них нет внутренних данных, которые не должны выходить за пределы компании

### Модуль 5: Автоматизация и результат (30 мин)

**Цель модуля:** Студент создаёт библиотеку skills для повторяющихся задач, настраивает workflow и измеряет экономию времени.

**Вклад в сквозной проект:** Финальная библиотека skills + автоматизированные workflow + Time Audit "после"

**Урок 5.1 — Skills: шаблоны для повторяющихся задач** (7 мин)
- Что такое skill: сохранённая инструкция для задачи, которую вы делаете регулярно
- Примеры skills: "еженедельный отчёт", "подготовка к встрече", "обработка входящих документов"
- Как создать skill: взять задачу из модулей 2-4, которая получилась хорошо → сохранить как шаблон
- Демонстрация: skill "Пятничный отчёт" — одна команда → готовый отчёт в Google Docs каждую пятницу

**Урок 5.2 — Workflow: связываем skills в цепочки** (8 мин)
- Workflow = несколько skills, которые запускаются вместе или последовательно
- Пример workflow "Конец месяца": собрать данные из Sheets → сформировать отчёт в Docs → подготовить презентацию → список action items
- Как организовать CLAUDE.md для workflow: секции по типу задач, ссылки на папки и шаблоны
- Антипаттерн: создавать skills для задач, которые вы делаете раз в год

**Урок 5.3 — Time Audit "после" и измерение результата** (5 мин)
- Вернуться к Time Audit "до" из модуля 1
- Замерить те же 3-5 задач с использованием Cowork + MCP
- Посчитать экономию: "Отчёт: было 2 часа → стало 25 минут", "Подготовка к встрече: было 40 минут → стало 10 минут"
- Как представить результат руководству или коллегам (конкретные цифры, не "стало быстрее")

**Урок 5.4 — Практика: финализация AI-офиса** (10 мин)
- **Задание на своих данных:**
  1. Создать минимум 3 skills на основе задач из модулей 2-4
  2. Собрать один workflow из 2+ skills
  3. Заполнить Time Audit "после" — замерить экономию на 3-5 задачах
  4. Обновить CLAUDE.md с учётом всех созданных skills и workflow
- Чеклист финального AI-офиса: Cowork настроен ✓ MCP подключён ✓ CLAUDE.md актуален ✓ Skills созданы ✓ Workflow работает ✓ Time Audit заполнен ✓

## Итоговый артефакт курса

Студент завершает курс с работающим **персональным AI-офисом**:

1. **Claude Cowork + Google Workspace MCP** — настроенная среда, где Claude работает напрямую с Google Docs, Sheets, Slides
2. **CLAUDE.md** — файл с контекстом должности, правилами и ограничениями
3. **Библиотека skills** (минимум 3) — готовые шаблоны для повторяющихся задач
4. **Workflow** (минимум 1) — цепочка из нескольких skills
5. **Time Audit (до/после)** — измеренная экономия времени на 3-5 реальных рабочих задачах
