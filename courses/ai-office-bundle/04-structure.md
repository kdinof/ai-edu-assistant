# AI Office Bundle

## Паспорт курса

| Параметр | Значение |
|----------|----------|
| Длительность | 3 часа |
| Формат | Видео-скринкасты (5-10 мин/урок) + текстовый дубль |
| Уровень | Начинающий |
| Пререквизиты | Базовые офисные навыки (почта, документы, таблицы). Опыт с AI не требуется |
| Основной инструмент | Claude (claude.ai) |
| Сквозной проект | Персональная AI-система для своих рабочих задач |
| Финальный артефакт | 5+ настроенных Claude Skills + измеренная экономия времени на реальных задачах |

---

## Сквозной проект

Студент строит персональную AI-систему пошагово: от первого запроса к Claude до библиотеки рабочих Skills с замеренной экономией времени. Каждый модуль расширяет набор инструментов и усложняет тип задач.

| Модуль | Вклад в сквозной проект |
|--------|------------------------|
| 1 | Первый рабочий документ, созданный с Claude. Понимание интерфейса и базовых промптов |
| 2 | 3 шаблона промптов для регулярных документов (письма, отчёты, протоколы) |
| 3 | Аналитическая сводка из своих рабочих данных (таблица или документ) |
| 4 | 2-3 рабочих Claude Skill, собранных из лучших промптов модулей 2-3 |
| 5 | Полная система: библиотека Skills + замер Time Saved + план развития |

## Структура курса

### Модуль 1: Первый результат за 5 минут (30 мин)

**Цель модуля:** Студент получает первый полезный результат с Claude и понимает базовые принципы работы с AI-ассистентом.

**Вклад в сквозной проект:** Первый рабочий документ — отправная точка для всей системы.

**Урок 1.1 — Знакомство с Claude** (7 мин)
- Что такое Claude и чем он отличается от поисковика: не ищет информацию, а помогает думать и создавать
- Интерфейс claude.ai: чат, загрузка файлов, история разговоров
- Демонстрация: создаём рабочее письмо за 2 минуты — от запроса до готового результата
- Главная ошибка новичков: писать "напиши письмо" без контекста (кому, зачем, какой тон)

**Урок 1.2 — Анатомия хорошего запроса** (8 мин)
- Формула запроса: Роль + Задача + Контекст + Формат = результат
- Пример: "Ты — ассистент руководителя. Напиши ответ на жалобу клиента. Контекст: клиент получил товар с опозданием на 3 дня. Тон: вежливый, с извинением и конкретным решением"
- Антипаттерн: "Сделай красиво" — Claude не читает мысли, ему нужна конкретика
- Итеративная доработка: как уточнять результат ("сделай короче", "добавь цифры", "измени тон на более формальный")

**Урок 1.3 — Загрузка файлов: Claude читает ваши документы** (8 мин)
- Какие файлы понимает Claude: PDF, DOCX, TXT, CSV, Excel, изображения
- Демонстрация: загружаем рабочий документ и просим Claude извлечь ключевые пункты
- Лимиты: размер файлов, количество за раз, что делать с очень длинными документами
- **Безопасность данных:** Что можно загружать в Claude: рабочие документы без персональных данных клиентов. Что нельзя: пароли, паспортные данные, номера банковских карт, конфиденциальные финансовые отчёты. Правило: если сомневаешься — не загружай

**Урок 1.4 — Практика: мой первый документ с Claude** (7 мин)
- **Задание на своих данных:** Создай с Claude документ, который тебе нужен прямо сейчас — рабочее письмо, ответ клиенту, заметку со встречи или короткий отчёт
- Чеклист: (1) Использовал формулу запроса, (2) Уточнил результат хотя бы 1 раз, (3) Результат можно использовать в работе
- **Безопасность данных:** Перед отправкой проверь — нет ли в тексте персональных данных клиентов или паролей

### Модуль 2: Документы на автопилоте (40 мин)

**Цель модуля:** Студент умеет создавать шаблоны промптов для регулярных документов и получать результат нужного качества с первого-второго запроса.

**Вклад в сквозной проект:** 3 шаблона промптов для документов, которые студент пишет регулярно.

**Урок 2.1 — Отчёты и протоколы** (8 мин)
- Промпт-шаблон для отчёта: структура (заголовок, период, ключевые метрики, выводы, следующие шаги)
- Демонстрация: из заметок со встречи → протокол за 3 минуты (было: 30 минут ручной работы)
- Как задавать формат: "Оформи как таблицу", "Раздели на секции с заголовками", "Добавь нумерованный список действий"
- Антипаттерн: отправлять Claude "сырые" заметки без указания, что именно нужно на выходе

**Урок 2.2 — Деловые письма и коммуникации** (8 мин)
- Шаблон для писем: адресат + цель + контекст + тон + ограничения по длине
- Примеры тонов: формальное письмо партнёру, дружелюбный ответ клиенту, внутреннее сообщение команде
- Демонстрация: пишем ответ на сложное письмо клиента — Claude предлагает 2 варианта тона
- Как адаптировать тон для узбекского бизнес-контекста: уважительная форма обращения, упоминание партнёрства

**Урок 2.3 — Редактирование и переписывание** (8 мин)
- Claude как редактор: "Сократи до 200 слов", "Упрости язык для неспециалиста", "Найди ошибки и предложи исправления"
- Демонстрация: берём длинный черновик отчёта → Claude сокращает в 3 раза, сохраняя суть
- Переписывание между форматами: email → пост для Telegram-канала, протокол → краткая сводка для руководства
- **Безопасность данных:** Как анонимизировать примеры — замена реальных имён клиентов на "Клиент А", реальных сумм на "XXX" перед загрузкой в Claude

**Урок 2.4 — Проверка качества: не доверяй слепо** (8 мин)
- Когда Claude ошибается: устаревшие данные, выдуманные факты, неточные цифры
- Правило проверки: факты и цифры в AI-output всегда проверяй вручную. Claude генерирует текст, а не истину
- Чеклист проверки документа: (1) Факты корректны? (2) Цифры совпадают с источником? (3) Тон соответствует? (4) Нет лишнего?
- Демонстрация: Claude пишет отчёт с "красивой" статистикой — проверяем, находим неточность, исправляем

**Урок 2.5 — Практика: мои шаблоны документов** (8 мин)
- **Задание на своих данных:** Выбери 3 типа документов, которые ты пишешь регулярно (письмо, отчёт, протокол, заявка). Для каждого создай промпт-шаблон с формулой из урока 1.2
- Чеклист: (1) Каждый шаблон содержит роль + задачу + контекст + формат, (2) Протестировал каждый шаблон — результат пригоден для работы, (3) Сохранил шаблоны в отдельный чат или заметку
- **Безопасность данных:** В шаблонах используй placeholder'ы вместо реальных данных: "[ИМЯ КЛИЕНТА]", "[СУММА]"

### Модуль 3: Данные без формул (40 мин)

**Цель модуля:** Студент умеет загружать таблицы и документы в Claude, извлекать ключевые данные и получать аналитические сводки без знания формул и макросов.

**Вклад в сквозной проект:** Аналитическая сводка из своих рабочих данных — первый опыт анализа с AI.

**Урок 3.1 — Таблицы в Claude: CSV и Excel** (8 мин)
- Как загрузить таблицу: CSV, Excel (XLSX) — просто перетащить в чат
- Что Claude видит в таблице: заголовки, типы данных, количество строк
- Демонстрация: загружаем таблицу продаж → Claude описывает структуру и предлагает вопросы для анализа
- Ограничения: очень большие таблицы (10 000+ строк) лучше разбить или загрузить сводку

**Урок 3.2 — Извлечение и сводки** (8 мин)
- Типовые запросы к таблицам: "Покажи топ-5 по выручке", "Сравни показатели за январь и февраль", "Найди аномалии"
- Демонстрация: таблица расходов отдела → Claude формирует сводку: общая сумма, топ-3 статьи, тренд по месяцам
- Как просить визуализацию: "Опиши тренд словами", "Сделай текстовую таблицу с итогами"
- Антипаттерн: загружать таблицу и писать "проанализируй" — без конкретного вопроса результат будет поверхностным

**Урок 3.3 — Длинные документы: PDF и отчёты** (8 мин)
- Сила Claude: контекстное окно 200K+ токенов — может прочитать документ на 100+ страниц целиком
- Типовые запросы: "Выдели ключевые пункты", "Найди все упоминания бюджета", "Сравни раздел 3 и раздел 7"
- Демонстрация: загружаем PDF-отчёт на 40 страниц → Claude извлекает 5 ключевых выводов за 30 секунд
- Работа с несколькими документами: загрузить 2-3 файла и попросить сравнить или объединить

**Урок 3.4 — Аналитические вопросы: что спрашивать у данных** (8 мин)
- Фреймворк вопросов: Что? (описание) → Почему? (причины) → Что дальше? (рекомендации)
- Примеры: "Какие продукты показали рост >20%?" → "Почему категория X упала?" → "Что рекомендуешь изменить?"
- Демонстрация: от сырой таблицы к аналитической записке для руководства за 10 минут
- **Безопасность данных:** Правила работы с таблицами: удалить столбцы с ФИО, ИНН, телефонами клиентов перед загрузкой. Использовать агрегированные данные (итоги по отделам) вместо построчных данных с персональной информацией

**Урок 3.5 — Практика: сводка из моих данных** (8 мин)
- **Задание на своих данных:** Загрузи свою рабочую таблицу или отчёт (таблица продаж, бюджет отдела, отчёт за квартал) и получи аналитическую сводку
- Чеклист: (1) Удалил персональные данные перед загрузкой, (2) Задал конкретный вопрос (не просто "проанализируй"), (3) Получил сводку с цифрами, которую можно показать руководителю
- **Безопасность данных:** Перед загрузкой пройдись по столбцам — удали или замени всё, что содержит персональные данные

### Модуль 4: Skills — промпт становится инструментом (35 мин)

**Цель модуля:** Студент умеет создавать Claude Skills — превращать лучшие промпты в повторяемые инструменты, которые дают стабильный результат каждый раз.

**Вклад в сквозной проект:** 2-3 рабочих Claude Skill для задач из модулей 2-3.

**Урок 4.1 — Что такое Claude Skills** (7 мин)
- Skill = сохранённая инструкция, которую Claude выполняет каждый раз одинаково хорошо
- Разница: промпт в чате (разовый) vs Skill (повторяемый инструмент)
- Демонстрация: промпт "напиши протокол встречи" → Skill "Протокол встречи", который каждый раз выдаёт протокол в нужном формате с нужной структурой
- Где найти Skills: Настройки профиля → Skills в интерфейсе Claude

**Урок 4.2 — Создаём первый Skill** (8 мин)
- Анатомия Skill: название, описание, инструкции (роль, задача, формат output, ограничения)
- Пошагово: берём лучший промпт из модуля 2 → оформляем как Skill
- Демонстрация: Skill "Деловое письмо" — принимает тему и контекст, выдаёт готовое письмо в корпоративном стиле
- Тестирование: запускаем Skill 3 раза с разными вводными — результат должен быть стабильно хорошим

**Урок 4.3 — Skills для анализа данных** (7 мин)
- Skill для таблиц: "Аналитическая сводка" — загрузи таблицу, получи сводку с ключевыми метриками, трендами и рекомендациями
- Skill для документов: "Выжимка из отчёта" — загрузи PDF, получи 5 ключевых пунктов + список действий
- Демонстрация: создаём Skill "Еженедельный отчёт из таблицы продаж" — каждую неделю загружаем новую таблицу, получаем отчёт в одном формате
- **Безопасность данных:** Не включай в инструкции Skill конфиденциальные данные (названия клиентов, суммы контрактов). Skill — это шаблон, данные подаются отдельно каждый раз

**Урок 4.4 — Отладка и улучшение Skills** (6 мин)
- Когда Skill работает не так: проверь инструкции — достаточно ли контекста? Есть ли примеры формата?
- Приём: добавь в Skill секцию "Формат output" с конкретным примером результата
- Приём: добавь ограничения ("Не добавляй информацию, которой нет во входных данных", "Максимум 300 слов")
- Итеративный подход: запустил → проверил → уточнил инструкции → запустил снова

**Урок 4.5 — Практика: мои рабочие Skills** (7 мин)
- **Задание на своих данных:** Возьми 2-3 лучших промпта из модулей 2-3 и преврати каждый в Claude Skill. Протестируй каждый Skill на реальной задаче
- Чеклист: (1) Skill имеет понятное название, (2) Инструкции включают роль + формат + ограничения, (3) Протестировал 2+ раза — результат стабильный, (4) Skill решает задачу, которую ты делаешь регулярно (минимум раз в неделю)
- **Безопасность данных:** Проверь инструкции Skills — убедись, что в них нет чувствительных данных

### Модуль 5: Моя AI-система (35 мин)

**Цель модуля:** Студент собирает все навыки и Skills в персональную AI-систему, замеряет реальную экономию времени и получает план развития.

**Вклад в сквозной проект:** Финальная сборка: библиотека Skills + замер Time Saved + план расширения.

**Урок 5.1 — Собираем библиотеку Skills** (8 мин)
- Аудит: какие Skills созданы в модуле 4? Какие задачи остались без Skill?
- Категории Skills для офиса: документы, коммуникации, анализ данных, подготовка к встречам
- Демонстрация: организуем библиотеку — 5+ Skills с понятными названиями, каждый для конкретной регулярной задачи
- Приём: для каждого Skill запиши, когда его использовать (триггер: "каждый понедельник", "при получении отчёта", "перед встречей")

**Урок 5.2 — Замер Time Saved** (8 мин)
- Метод замера: выбери 3 задачи, которые делаешь регулярно. Засеки время без Claude и с Claude
- Пример: подготовка протокола — было 30 минут → стало 5 минут с Claude. Экономия: 25 минут на каждой встрече
- Как считать: задача X за неделю × экономия минут = часы, освобождённые за месяц
- Демонстрация: заполняем таблицу Time Saved для 3 задач — видим реальную экономию в часах за месяц

**Урок 5.3 — Продвинутые приёмы** (7 мин)
- Projects в Claude: группируй связанные разговоры по проекту (подготовка к конференции, квартальный отчёт)
- Контекст проекта: добавь документы в Project — Claude будет учитывать их во всех разговорах внутри проекта
- Цепочки запросов: результат одного запроса → вход для следующего (данные → сводка → письмо руководству)
- Что дальше: Claude постоянно обновляется — следи за новыми возможностями на claude.ai

**Урок 5.4 — Практика: моя AI-система** (7 мин)
- **Задание на своих данных:** (1) Проверь библиотеку Skills — минимум 5 рабочих Skill. (2) Замерь Time Saved на 3 реальных задачах. (3) Запиши план: какие ещё задачи автоматизировать в следующем месяце
- Чеклист: (1) 5+ Skills в библиотеке, каждый для регулярной задачи, (2) Таблица Time Saved для 3 задач заполнена, (3) План развития на следующий месяц — минимум 2 новых направления
- **Безопасность данных:** Финальная проверка: ни один Skill не содержит персональные данные или пароли в инструкциях

**Урок 5.5 — Тест: проверь свои знания** (5 мин)
- Тест на ключевые понятия курса
- Получение результата

## Итоговый артефакт курса

Студент завершает курс с:

1. **Библиотека Claude Skills (5+)** — персональные инструменты для регулярных рабочих задач: документы, письма, анализ данных, отчёты. Собрана пошагово через модули 1-5
2. **Таблица Time Saved** — измеренная экономия времени на 3+ реальных задачах с конкретными цифрами "было → стало"
3. **3 шаблона промптов** — проверенные шаблоны для типовых документов (из модуля 2)
4. **План развития** — список задач для автоматизации в следующем месяце
