# PRD: AI Content Pipeline

**Дата:** 2026-02-23 (обновлено 2026-02-26)
**Источник:** Встреча команды 23 февраля + переписка с Антоном 26 февраля
**Статус:** Draft

---

## Проблема

1. **Долгий time-to-market контента.** Создание курсов занимает много времени: спикеры, съёмки, монтаж, загрузка на платформу. Команда production использует GPT и Keygen точечно, но нет единого пайплайна.

2. **Контент быстро устаревает.** AI-инструменты обновляются постоянно, а видеоконтент сложно и дорого переснимать.

3. **Много ручной работы при загрузке.** Даже после генерации текста в GPT, продюсерам нужно вручную оформлять и загружать контент на платформу.

---

## Видение

AI-пайплайн, который автоматизирует полный цикл создания образовательного контента — от темы до готового курса на платформе.

Начинаем с прототипа на агентах Claude Code (vendor lock, работает на ПК), валидируем пайплайн, затем переписываем в автономную серверную систему с UI.

---

## Стратегическое значение

Компания готова инвестировать 3-6 месяцев. Это не одноразовый эксперимент, а долгосрочное направление.

**Почему важно:**
1. **Инвесторы** — AI-продукт внутри EdTech платформы повышает привлекательность для инвестиций
2. **Клиенты** — можно продавать как фичу: "у нас есть AI, который помогает создавать/проходить курсы"
3. **Госка/фонды** — AI в образовании — приоритетная тема для грантов и госпрограмм

---

## Пайплайн

```
Тема → PRD образовательного продукта → Структура курса → Контент (тексты) → Наполнение (картинки, видео) → Вёрстка под платформу
```

### Deep Research

На этапе структуры подключается deep research (Perplexity / OpenAI Deep Research):
- Анализ существующих курсов по теме на рынке
- Сбор актуальной информации об инструментах
- Обеспечение актуальности контента (решает проблему устаревания)

### Агенты

1. **Input:** Тема курса + описание целевой аудитории
2. **Агент 1 — PRD:** Генерирует PRD для образовательного продукта (описание, цели, ЦА). Задаёт уточняющие вопросы.
3. **Агент 2 — Структура:** Deep research + создание структуры (модули → темы → уроки)
4. **Агент 3 — Контент:** Генерирует текстовый контент под каждый урок
5. **Агент 4 — Тесты и задания:** Создаёт проверочные задания

### База знаний

Все предыдущие курсы OSNOVA загружаются как reference — система знает формат, стиль и стандарты контента.

---

## Метрики

- **Первая метрика:** Скорость создания контента (time-to-market)
- **Вторая метрика:** Качество контента (оценка командой + глубина прохождения студентами)

---

## Roadmap (3 фазы)

Характер работы — R&D: экспериментируем, что-то пойдёт быстро, где-то придётся покопаться. Ниже — концептуальные фазы, не жёсткие дедлайны.

### Фаза 1: Прототип на Claude Code (vendor lock, на ПК)

**Цель:** Валидация пайплайна. Проверить, что агенты выдают контент нужного качества.

**Что делаем:**
- Агентская система внутри Claude Code: тема → PRD → структура → контент → тесты
- Deep research для актуальности
- База знаний из существующих курсов OSNOVA
- Перевод курсов на узбекский язык (исследовать API для качественного перевода на узбекский — у Claude нет качественного перевода из коробки)
- Тестирование на реальных курсах с командой production
- Всё работает на ПК, привязано к Claude Code

**Результат фазы:** Работающий пайплайн, команде нравится результат, понятно что работает, а что нет.

**Команда:** Кирилл (AI/продукт), методолог (парт-тайм, валидация контента)
**Бюджет:** ~$20/мес токены + $10-20 research

### Фаза 2: Автономная серверная система + evals (без vendor lock)

**Цель:** Переписать проверенный пайплайн в автономную агентскую систему на сервере. Параллельно запустить систему оценки качества.

**Что делаем:**
- Агентский бэкенд — не привязан к Claude Code, работает на сервере
- Базовый UI для доступа к системе (вход по логину/паролю, создание курсов через интерфейс)
- Выбор моделей без привязки к одному провайдеру
- **Evals (запускается параллельно):** определить критерии оценки качества контента — соответствие методологии, полнота, понятность, пригодность для студентов

**Результат фазы:** Серверная система с бэкендом и базовым интерфейсом + понимание, как оцениваем качество output'а пайплайна.

**Команда:** Кирилл + бэкенд-разработчик (парт-тайм), методолог (парт-тайм, критерии качества)

### Фаза 3: Качество + полноценный UI

**Цель:** Полноценный интерфейс для создания курсов. Фокус на качестве контента через evals.

**Что делаем:**
- Полноценный UI: регистрация, авторизация, создание контента на основе вводных
- Итерации по качеству на основе evals, фидбэка команды и метрик
- Автоматические evals в пайплайне (качество проверяется на каждом прогоне)

**Результат фазы:** Продукт, в который можно зайти, задать вводные и получить готовый текстовый образовательный контент с проверенным качеством.

**Команда:** Кирилл + фронтенд-разработчик + бэкенд-разработчик (парт-тайм)

---

## Гипотезы после фазы 3

После полноценного UI с текстовым контентом — пространство для экспериментов по обогащению output'а. Каждая гипотеза требует отдельной проверки экономики.

### Гипотеза 1: Вёрстка под конкретную LMS

Output пайплайна в формате, который можно быстро загрузить в LMS (SCORM, xAPI, или API конкретной платформы). Контент появляется в LMS без ручной вёрстки.

### Гипотеза 2: Картинки

AI-генерация иллюстраций, поиск подходящих изображений (Google Images), проверка соответствия контексту урока.

### Гипотеза 3: Аудио

Озвучка частей курса, запись подкаст-формата (как в NotebookLM). TTS-модели.

### Гипотеза 4: Видео

Самая сложная и дорогая часть. AI-аватары, скринкасты.

**Открытый вопрос:** Сходится ли экономика? AI-видео с аватаром vs. реальная съёмка человека со скринкастом. Учитывая, что скринкасты — основной формат, а спикер может говорить параллельно с записью экрана, преимущество AI-видео не очевидно. Нужен расчёт.

| Кирилл (AI/продукт) | + | + | + |

## Риски и ограничения

| Риск | Митигация |
|------|-----------|
| R&D-характер — сложно давать точные сроки | Фазы концептуальные, регулярные демо результатов |
| Контент устаревает | Deep research + быстрый пайплайн обновления |
| Качество генерации ниже ожиданий | Специализированная база знаний OSNOVA + фаза 3 про evaluation |
| Видео-генерация дорогая | Начать с текста, исследовать Claude + Remotion |
| Быстро меняющийся landscape AI | Фаза 2 убирает vendor lock — легко менять модели |

## Смежные инициативы

Параллельно на платформе планируются:

- **Sandbox (чат-бокс)** — практика с AI прямо на платформе (март, команда разработки)
- **Персонализация обучения** — адаптация курсов под параметры студента (backlog)

Эти направления могут использовать контент из пайплайна, но являются самостоятельными проектами.
